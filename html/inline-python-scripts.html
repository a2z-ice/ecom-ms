1. Overview

The BookStore platform's shell scripts use inline Python for JSON/YAML manipulation tasks that are impractical in pure bash.

All 18 snippets use only the Python standard library (json, sys) plus yaml (PyYAML) where YAML parsing is required. No third-party JSON tools like jq or yq are needed.

Location: up.sh:378-396 Pattern: YAML+JSON ConfigMap Condition: Only if csrf-ext-authz not in mesh config

Input: ConfigMap JSON from kubectl
Output: Modified ConfigMap piped to kubectl apply

Python Logic 
import sys, json, yaml
cm = json.load(sys.stdin)           # parse ConfigMap JSON
mesh = yaml.safe_load(              # parse embedded YAML
    cm['data']['mesh'])
if 'extensionProviders' not in mesh:
    mesh['extensionProviders'] = [] # init list if missing
mesh['extensionProviders'].append({ # add new provider
    'name': 'csrf-ext-authz',
    'envoyExtAuthzHttp': {
        'service': 'csrf-service.infra...',
        'port': 8080, ...
    }
})
cm['data']['mesh'] = yaml.dump(     # re-serialize YAML
    mesh, default_flow_style=False)
json.dump(cm, sys.stdout)           # output full ConfigMap

Why Python: The mesh config is YAML embedded inside a JSON ConfigMap. Bash cannot parse YAML, and jq cannot parse YAML either. Python with PyYAML handles both layers.

Location: register-schemas.sh:72-77 Pattern: File Processing Condition: Runs for each subject/schema pair in the SUBJECTS array

    schema = f.read().strip()

    'schemaType': 'JSON',

")

Input: JSON schema file on disk
Output: Schema Registry payload with escaped content

Python Logic (line-by-line) 
import json

# 1. Open the schema file
with open('schemas/order.json') as f:
    schema = f.read().strip()
    # schema = '{"type":"object",...}'

# 2. Wrap in Registry envelope
#    json.dumps handles all escaping:
#    - quotes become \"
#    - newlines become \n
#    - backslashes become \\
print(json.dumps({
    'schemaType': 'JSON',
    'schema': schema
}))
# Output: {"schemaType":"JSON",
#  "schema":"{\"type\":\"object\",...}"}

Why Python: This is the most complex snippet. The schema file content must be JSON-escaped (quotes, newlines, backslashes) and embedded as a string value inside another JSON object. This "JSON inside JSON" escaping is fragile or impossible with sed/awk, and bash has no built-in JSON escape function.

Expected Output:
Input file (schemas/order-created-value.json):

  "title": "Application Event — order.created",
  "properties": {
    "orderId": { "type": "string", "format": "uuid" },
    "userId": { "type": "string" },
    "items": { "type": "array", ... }
  }
}

{
  "schemaType": "JSON",
  "schema": "{\n  \"type\": \"object\",\n  \"title\": \"Application Event \\u2014 order.created\",\n  \"properties\": {\n    \"orderId\": { \"type\": \"string\", \"format\": \"uuid\" },\n    ..."

3. Pattern Catalog

Five reusable patterns covering all 18 inline Python snippets. Each includes a canonical example, when to use it, and common gotchas.

Pattern 1: JSON Field Extraction 10 usages

The most common pattern. Extracts a single field from a JSON API response.

Canonical Example

VALUE=$(curl -s http://api/endpoint \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('key', 'default'))" \
  2>/dev/null || echo "fallback")

When to Use

Any time you need a single value from a JSON response in a bash variable.

Variations

VariationPython ExpressionUsed In
Single field.get('status', 'UNKNOWN')Health checks (6 scripts)
Array lengthlen(json.load(sys.stdin))Book/cert counts (3 scripts)
Filtered countsum(1 for j in d.get('jobs',[]) if j['state']=='RUNNING')Flink job count
Sorted iterationfor s in sorted(subjects): print(s)Schema subject listing

Common Gotchas

  • Always include 2>/dev/null after the Python command to suppress parse errors on malformed input
  • Always include || echo "fallback" to handle cases where curl or Python fails entirely
  • Use .get('key', 'default') rather than ['key'] to avoid KeyError on missing fields

Pattern 2: YAML+JSON ConfigMap Edit 3 usages

Modifies an Istio ConfigMap that contains embedded YAML inside a JSON structure.

Canonical Example

kubectl get configmap NAME -n NAMESPACE -o json | python3 -c "
import sys, json, yaml
cm = json.load(sys.stdin)
mesh = yaml.safe_load(cm['data']['mesh'])
# ... modify mesh dict ...
cm['data']['mesh'] = yaml.dump(mesh, default_flow_style=False)
json.dump(cm, sys.stdout)
" | kubectl apply -f -

When to Use

When you need to modify Istio mesh configuration or any ConfigMap with embedded YAML.

Common Gotchas

  • Requires PyYAML (yaml module) -- verify availability: python3 -c "import yaml"
  • Use yaml.safe_load (not yaml.load) to avoid arbitrary code execution
  • Use default_flow_style=False to produce human-readable block-style YAML
  • Always check for the key's existence before appending to avoid duplicates

Pattern 3: NetworkPolicy JSON Patch 2 usages

Reads a NetworkPolicy, inserts a rule at a specific position, and reapplies it.

Canonical Example

kubectl get networkpolicy NAME -n NAMESPACE -o json | python3 -c "
import sys, json
pol = json.load(sys.stdin)
pol['spec']['egress'].insert(0, {
    'to': [{'podSelector': {'matchLabels': {'app': 'target-app'}}}],
    'ports': [{'port': 8080, 'protocol': 'TCP'}]
})
json.dump(pol, sys.stdout)
" | kubectl apply -f -

When to Use

When you need to programmatically add rules to an existing NetworkPolicy without replacing the entire spec.

Common Gotchas

  • insert(0, ...) places the rule at the front -- position matters for readability
  • Always store the JSON in a variable first and check for existence/duplicates before patching
  • The full resource (including metadata, apiVersion) must be preserved -- json.load/json.dump handles this

Pattern 4: Token Extraction 2 usages

Extracts JWT access tokens from Keycloak OIDC token responses.

Canonical Example

TOKEN=$(curl -sk -X POST \
  "https://idp.keycloak.net:30000/realms/bookstore/protocol/openid-connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password&client_id=ui-client&username=USER&password=PASS" \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('access_token',''))" \
  2>/dev/null || echo "")

When to Use

When you need to obtain a JWT from Keycloak for API testing in shell scripts.

Common Gotchas

  • JWTs contain dots, equals, and base64 characters that confuse bash -- always quote the variable
  • Use -sk with curl when the Keycloak endpoint uses a self-signed certificate
  • Check for empty token before using: [[ -n "$TOKEN" ]]

Pattern 5: File Read + JSON Escape 1 usage

Reads a file and embeds its content as a JSON-escaped string inside a wrapper object.

Canonical Example

payload=$(python3 -c "
import json
with open('path/to/schema.json') as f:
    schema = f.read().strip()
print(json.dumps({'schemaType': 'JSON', 'schema': schema}))
")

When to Use

When a file's content must be embedded as a string value inside a JSON payload (e.g., Schema Registry, configuration endpoints).

Common Gotchas

  • json.dumps() handles all escaping: quotes become \", newlines become \n, backslashes become \\
  • The file path is interpolated by bash -- ensure the path does not contain single quotes
  • Use .strip() to remove trailing newlines that could cause issues

4. Why Python Instead of jq/yq/bash

Python 3 is the only tool that satisfies all requirements: pre-installed, handles JSON+YAML, safe string escaping.

Alternative Limitation
jq Not installed on all systems; cannot parse YAML; requires separate installation via Homebrew/apt
yq Not standard; multiple incompatible versions exist (kislyuk/yq vs mikefarah/yq); different syntax
bash string ops Fragile for JSON; breaks on special characters, nested objects, arrays; no JSON escape function
sed / awk Cannot safely manipulate JSON arrays or nested structures; no understanding of JSON syntax
python3 Pre-installed on macOS and Linux; handles JSON+YAML natively; safe string escaping; predictable error handling

Key Advantage: Zero Additional Dependencies

Every macOS and modern Linux distribution ships with Python 3 and the json module. The yaml module (PyYAML) is needed only for the 3 ConfigMap manipulation snippets and is available in the project's Python environment (inventory-service).

This means any developer can run scripts/up.sh or scripts/smoke-test.sh without installing additional tools beyond what the OS provides.
