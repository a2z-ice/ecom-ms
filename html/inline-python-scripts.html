1. Overview

The BookStore platform's shell scripts in scripts/ use inline Python for JSON/YAML manipulation tasks that are impractical in pure bash.

All 14 snippets use only the Python standard library (json, sys) plus yaml (PyYAML) where YAML parsing is required. No third-party JSON tools like jq or yq are needed. Every snippet is idempotent and safe to re-run.

10 usages

scripts/up.sh Master bootstrap script -- creates/recovers the entire cluster 5 Python usages

1. Istio mesh config -- register csrf-ext-authz extensionProvider

Pattern: YAML-in-JSON ConfigMap Condition: Only if csrf-ext-authz not already present
Shell Pipe Chain 
kubectl get configmap istio \
  -n istio-system -o json \
| python3 -c "..." \
| kubectl apply -f -
Python Logic 
import sys, json, yaml
cm = json.load(sys.stdin)
mesh = yaml.safe_load(cm['data']['mesh'])
if 'extensionProviders' not in mesh:
    mesh['extensionProviders'] = []
mesh['extensionProviders'].append({
    'name': 'csrf-ext-authz',
    'envoyExtAuthzHttp': {
        'service': 'csrf-service.infra.svc.cluster.local',
        'port': 8080,
        'failOpen': True,
        'headersToUpstreamOnAllow': [],
        'includeRequestHeadersInCheck': [
            'authorization', 'x-csrf-token',
            'origin', 'referer'
        ],
    }
})
cm['data']['mesh'] = yaml.dump(
    mesh, default_flow_style=False)
json.dump(cm, sys.stdout)

Why Python: The mesh config is YAML embedded inside a JSON ConfigMap. Bash cannot parse YAML and jq cannot parse YAML. Python with PyYAML handles both layers.

2. Patch gateway-egress NetworkPolicy to allow csrf-service

Pattern: K8s Policy Array Patch Condition: Only if csrf-service rule not already present
Shell Pipe Chain 
_GW_EGRESS=$(kubectl get networkpolicy \
  gateway-egress -n infra -o json)
echo "$_GW_EGRESS" \
| python3 -c "..." \
| kubectl apply -f -
Python Logic 
import sys, json
pol = json.load(sys.stdin)
pol['spec']['egress'].insert(0, {
    'to': [{'podSelector': {'matchLabels':
        {'app': 'csrf-service'}}}],
    'ports': [{'port': 8080,
               'protocol': 'TCP'}]
})
json.dump(pol, sys.stdout)

Why Python: insert(0, ...) prepends a new egress rule at index 0 without disturbing the existing rules. kubectl patch strategic merge cannot insert at a specific array position.

3. Patch AuthorizationPolicy to add admin-tools namespace

Pattern: K8s Policy Array Patch Applied to: 4 policies (ecom-db, inventory-db, analytics-db, keycloak-db) 
echo "$_AP" | python3 -c "import sys,json;
p=json.load(sys.stdin);
p['spec']['rules'].append({'from':[{'source':{'namespaces':['admin-tools']}}]});
json.dump(p,sys.stdout)" | kubectl apply -f -

Purpose: Grants PgAdmin (in admin-tools namespace) access to all 4 database AuthorizationPolicies. Idempotent via upstream grep -q admin-tools check before calling Python.

4. Patch NetworkPolicy to allow admin-tools ingress

Pattern: K8s Policy Array Patch Applied to: 4 policies (ecom-db, inventory-db, analytics-db, keycloak-db) 
echo "$_NP" | python3 -c "import sys,json;
p=json.load(sys.stdin);
p['spec']['ingress'].append({'from':[{'namespaceSelector':
  {'matchLabels':{'kubernetes.io/metadata.name':'admin-tools'}}}]});
json.dump(p,sys.stdout)" | kubectl apply -f -

Purpose: The NetworkPolicy partner of Snippet 3. Both AuthorizationPolicy AND NetworkPolicy must allow admin-tools for PgAdmin to reach each DB.

4. Count RUNNING Flink streaming jobs (filter state)

Pattern: JSON Field Extraction (filtered count) Expected: 4 jobs 
FLINK_JOBS=$(curl -s --max-time 10 "http://localhost:32200/jobs/overview" 2>/dev/null \

3. Pattern Catalog

Three reusable patterns covering all 14 inline Python snippets. Each includes a canonical example, when to use it, and common gotchas.

Pattern 1: JSON Field Extraction 10 usages

The most common pattern. Extracts a single field (or array length, or filtered count) from a JSON API response.

Canonical Example

VALUE=$(curl -s http://api/endpoint 2>/dev/null \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('key', 'default'))" \
  2>/dev/null || echo "fallback")

When to Use

Any time you need a single value from a JSON response bound to a bash variable.

Variations Found in scripts/

VariationPython ExpressionUsed In
Single field.get('status', 'UNKNOWN')Debezium health (4 places)
Access token.get('access_token', '')smoke-test.sh (2 places)
Array lengthlen(json.load(sys.stdin))Book count, cert count (3 places)
Filtered countsum(1 for j in d.get('jobs',[]) if j['state']=='RUNNING')Flink jobs (1 place)

Common Gotchas

  • Always append 2>/dev/null after the Python command to suppress parse errors on malformed input
  • Always append || echo "fallback" to handle cases where curl or Python fails entirely
  • Use .get('key', 'default') rather than ['key'] to avoid KeyError on missing fields
  • Quote the resulting bash variable when used with [[ ... ]] — JWT tokens contain special chars

Pattern 2: Kubernetes Policy Array Patching 3 usages

Reads a live Kubernetes policy resource, appends or inserts a rule, and reapplies it via kubectl apply.

Canonical Example (append)

echo "$POLICY_JSON" | python3 -c "
import sys, json
p = json.load(sys.stdin)
p['spec']['rules'].append({
    'from': [{'source': {'namespaces': ['admin-tools']}}]
})
json.dump(p, sys.stdout)
" | kubectl apply -f -

Canonical Example (insert at index)

echo "$POLICY_JSON" | python3 -c "
import sys, json
pol = json.load(sys.stdin)
pol['spec']['egress'].insert(0, {
    'to': [{'podSelector': {'matchLabels': {'app': 'csrf-service'}}}],
    'ports': [{'port': 8080, 'protocol': 'TCP'}]
})
json.dump(pol, sys.stdout)
" | kubectl apply -f -

When to Use

When you must programmatically add a rule to an existing policy without replacing the entire spec (preserves other rules, metadata, status).

Common Gotchas

  • Always do an idempotency check BEFORE calling Python: grep -q admin-tools. Calling Python unconditionally creates duplicate rules.
  • insert(0, ...) prepends; append(...) adds to the end. Choose based on policy evaluation order requirements.
  • The full resource (apiVersion, kind, metadata, status) is passed through untouched — json.load/json.dump preserves everything.
  • kubectl apply -f - reads stdin; pipe the Python output directly.

Pattern 3: YAML-in-JSON ConfigMap Mutation 1 usage

Modifies an Istio ConfigMap whose data.mesh field is a YAML document (as a string) inside a JSON ConfigMap structure.

Canonical Example

kubectl get configmap istio -n istio-system -o json | python3 -c "
import sys, json, yaml
cm = json.load(sys.stdin)           # parse outer JSON (ConfigMap)
mesh = yaml.safe_load(cm['data']['mesh'])  # parse inner YAML string
# ... modify mesh dict ...
cm['data']['mesh'] = yaml.dump(mesh, default_flow_style=False)
json.dump(cm, sys.stdout)
" | kubectl apply -f -

When to Use

Exclusively for Istio's istio-system/istio ConfigMap — that's the one case in this project where YAML is embedded inside a JSON ConfigMap.

Common Gotchas

  • Requires PyYAML (yaml module). It's shipped with the Python in this project's environment.
  • Use yaml.safe_load (not yaml.load) to avoid arbitrary Python code execution.
  • Use default_flow_style=False to keep the output human-readable (block style, not JSON-ish flow).
  • Always check if the change already exists before appending (via grep upstream or if ... not in in Python).

4. Why Python Instead of jq/yq/bash

Python 3 is the only tool that satisfies all requirements: pre-installed, handles JSON+YAML, safe string escaping.

Alternative Limitation
jq Not installed on all systems; cannot parse YAML; requires separate installation via Homebrew/apt
yq Not standard; multiple incompatible versions exist (kislyuk/yq vs mikefarah/yq); different syntax
bash string ops Fragile for JSON; breaks on special characters, nested objects, arrays; no JSON escape function
sed / awk Cannot safely manipulate JSON arrays or nested structures; no understanding of JSON syntax
python3 Pre-installed on macOS and Linux; handles JSON+YAML natively; safe string escaping; predictable error handling

Key Advantage: Zero Additional Dependencies

Every macOS and modern Linux distribution ships with Python 3 and the json module. The yaml module (PyYAML) is needed only for the 1 Istio mesh-config snippet and is available in the project's Python environment.

Note: scripts/up.sh verifies python3 is on PATH in its pre-flight check alongside kind, kubectl, docker, helm, and istioctl.

This means any developer can run scripts/up.sh or scripts/smoke-test.sh without installing additional tools beyond what the OS provides.
