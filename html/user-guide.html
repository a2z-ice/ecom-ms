1. Prerequisites & Initial Setup

1Configure /etc/hosts

Add the following entries to your /etc/hosts file so that the platform hostnames resolve to localhost:

127.0.0.1  idp.keycloak.net myecom.net api.service.net
Required: Without these entries, the browser will not be able to reach the gateway endpoints. On macOS, edit with sudo nano /etc/hosts.

2Bootstrap the Cluster

Run the smart startup script. It auto-detects whether to perform a fresh bootstrap, recovery, or health check:

bash scripts/up.sh

For a full teardown and rebuild from scratch:

bash scripts/up.sh --fresh
First run: The initial bootstrap takes 5-10 minutes. It creates a kind cluster, installs Istio, deploys all infrastructure, imports the Keycloak realm, and registers Debezium connectors.

3Trust the Self-Signed CA

All gateway endpoints use HTTPS with a self-signed CA managed by cert-manager. To avoid browser warnings:

bash scripts/trust-ca.sh --install

This extracts the CA certificate from the cluster and adds it to the macOS Keychain. For curl commands, use the -sk flag.

4Verify the Stack

Run the smoke test to confirm all services are healthy:

bash scripts/smoke-test.sh

For a more comprehensive check including Kafka consumer lag and Debezium health:

bash scripts/sanity-test.sh

2. Book Store Application (Customer Experience)

1Open the Store

Navigate to https://myecom.net:30000 in your browser. You will see the book catalog with cover images, prices, and real-time stock availability badges.

2Login via Keycloak

Click the Login button in the navigation bar. You will be redirected to the Keycloak login page at https://idp.keycloak.net:30000/realms/bookstore/...

Customer credentials: Username: user1 — Password: CHANGE_ME

The login uses OIDC Authorization Code Flow with PKCE (S256). Tokens are stored in memory only and never persisted to localStorage.

3Browse & Add to Cart

After login, your name appears in the navbar alongside a Logout button. Each book displays a stock status badge:

  • In Stock — Sufficient inventory available
  • Only X left — Low stock warning (fewer than 5 units)
  • Out of Stock — Button disabled, cannot add to cart

Click Add to Cart on any in-stock book. The cart badge in the navbar updates with the item count.

4Checkout

Click the cart icon to open the Cart page. It displays items with quantities, prices, and per-item stock availability. Click Checkout to complete the purchase.

What happens behind the scenes:
  1. E-Commerce Service validates the cart and JWT token
  2. Stock is reserved via mTLS call to Inventory Service
  3. Order is persisted and an order.created event is published to Kafka
  4. Debezium captures the DB change, Flink transforms it into the analytics schema
  5. Cart is cleared after successful checkout
Checkout button: Only enabled when all items in the cart have sufficient stock. If any item exceeds available inventory, a warning banner appears and checkout is blocked.

5Guest Cart (No Login Required)

Guest users can browse the catalog and add items to cart without logging in. The guest cart is stored in localStorage under the key bookstore_guest_cart.

When a guest later logs in, all guest cart items are automatically merged into the server-side cart. Checkout always requires authentication.

3Force Renewal (Advanced)

To manually trigger certificate renewal via the API, you need a Kubernetes ServiceAccount token:

# Get a ServiceAccount token
TOKEN=$(kubectl create token bookstore-certs -n cert-dashboard --duration=10m)

# Trigger renewal via API
curl -X POST http://localhost:32600/api/renew \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"name":"bookstore-gateway-cert","namespace":"infra"}'

The dashboard shows SSE live progress through phases: deleting-secret → waiting-issuing → issued → ready → complete.

12. API Quick Reference

1Obtain a JWT Token

Use the Keycloak token endpoint to get an access token via Resource Owner Password Credentials grant:

TOKEN=$(curl -sk -X POST \
  "https://idp.keycloak.net:30000/realms/bookstore/protocol/openid-connect/token" \
  -d "grant_type=password&client_id=ui-client&username=user1&password=CHANGE_ME" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
Note: All gateway endpoints use HTTPS with a self-signed certificate. Always use the -sk flags with curl to skip certificate verification.

2Public Endpoints (No Authentication)

# List all books
curl -sk https://api.service.net:30000/ecom/books | jq .

# Search books by keyword
curl -sk https://api.service.net:30000/ecom/books/search?q=Python | jq .

# Get a specific book by ID
curl -sk https://api.service.net:30000/ecom/books/{book-uuid} | jq .

# Inventory health check
curl -sk https://api.service.net:30000/inven/health

# Get stock for a specific book
curl -sk https://api.service.net:30000/inven/stock/{book-uuid} | jq .

# Bulk stock lookup
curl -sk "https://api.service.net:30000/inven/stock/bulk?book_ids=uuid1,uuid2" | jq .

3Authenticated Endpoints (JWT Required)

# Get cart contents
curl -sk -H "Authorization: Bearer $TOKEN" \
  https://api.service.net:30000/ecom/cart | jq .

# Add item to cart
curl -sk -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"bookId":"<uuid>","quantity":1}' \
  https://api.service.net:30000/ecom/cart

# Checkout (with idempotency key)
curl -sk -X POST -H "Authorization: Bearer $TOKEN" \
  -H "Idempotency-Key: unique-key-123" \
  https://api.service.net:30000/ecom/checkout

4Admin Endpoints (Requires admin Role)

# Get an admin token

13. Troubleshooting & Common Tasks

1Check Cluster Health

# Quick smoke test (pods + HTTP routes + Kafka + Debezium)
bash scripts/smoke-test.sh

# Comprehensive sanity check
bash scripts/sanity-test.sh

2View Pod Logs

# E-Commerce Service (Spring Boot)
kubectl logs -n ecom deploy/ecom-service -f

# Inventory Service (FastAPI)
kubectl logs -n inventory deploy/inventory-service -f

# Keycloak
kubectl logs -n identity deploy/keycloak -f

# Debezium Servers
kubectl logs -n infra deploy/debezium-server-ecom -f
kubectl logs -n infra deploy/debezium-server-inventory -f

3Access Databases Directly

# E-Commerce database
kubectl exec -n ecom -it ecom-db-1 -- psql -U ecom

# Inventory database
kubectl exec -n inventory -it inventory-db-1 -- psql -U inventory

# Analytics database (CDC target)
kubectl exec -n analytics -it analytics-db-1 -- psql -U analytics

4Kafka Operations

# List all topics
kubectl exec -n infra deploy/kafka -- kafka-topics --bootstrap-server localhost:9092 --list

# List consumer groups
kubectl exec -n infra deploy/kafka -- kafka-consumer-groups --bootstrap-server localhost:9092 --list

# Check consumer lag
kubectl exec -n infra deploy/kafka -- kafka-consumer-groups \
  --bootstrap-server localhost:9092 --describe --all-groups

5Reset User Password

6Recovery After Docker Desktop Restart

After restarting Docker Desktop, the cluster enters a degraded state. The smart startup script auto-detects and handles recovery:

bash scripts/up.sh
Recovery steps performed automatically:
  1. ztunnel restart (Istio Ambient mesh HBONE re-plumbing)
  2. Pod rolling restart in dependency order (DBs first, then apps)
  3. Debezium connector re-registration (Kafka topics are lost on restart)

7Teardown

# Delete cluster (keeps data directory)
bash scripts/down.sh

# Delete cluster + wipe all persistent data
bash scripts/down.sh --data

# Delete cluster + data + Docker images
bash scripts/down.sh --all
