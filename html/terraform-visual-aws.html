1 S3 Backend + DynamoDB

# 00-backend.tf
terraform {
  backend "s3" {
    bucket         = "bookstore-terraform-state"
    key            = "bookstore/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "bookstore-terraform-locks"
    encrypt        = true
  }
}
Connects to: All Terraform operations
Local equivalent: Local .tfstate file (no locking)
Purpose: Stores Terraform state remotely in S3 so the entire team shares a single source of truth. DynamoDB provides distributed locking to prevent two people from running terraform apply simultaneously and corrupting state. Encryption at rest protects sensitive data (DB passwords, endpoints) stored in the state file.

2 VPC (Virtual Private Cloud)

# 03-vpc.tf
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "~> 5.16"

  name = "bookstore-dev-vpc"
  cidr = "10.0.0.0/16"

  azs = ["us-east-1a", "us-east-1b", "us-east-1c"]

  public_subnets   = ["10.0.1.0/24", "10.0.2.0/24", "10.0.3.0/24"]
  private_subnets  = ["10.0.10.0/24", "10.0.20.0/24", "10.0.30.0/24"]
  database_subnets = ["10.0.100.0/24", "10.0.101.0/24", "10.0.102.0/24"]

  enable_nat_gateway  = true
  enable_dns_hostnames = true
  enable_dns_support   = true
}
Connects to: EKS, Aurora, MSK, ElastiCache, ALB
Local equivalent: Docker bridge network (kind)
Purpose: The VPC is the isolated network that contains every AWS resource. It provides 65,536 IP addresses (/16 CIDR) divided into three tiers: public (internet-facing), private (compute), and database (most restricted). On kind, Docker provides a flat bridge network; on AWS, explicit subnets enforce network segmentation and defense in depth.

3 Public Subnets + Internet Gateway + NAT Gateway

# Inside module "vpc" { ... }

# Public subnets with EKS ALB discovery tags
public_subnets = ["10.0.1.0/24", "10.0.2.0/24", "10.0.3.0/24"]
public_subnet_tags = {
  "kubernetes.io/role/elb" = 1
  "kubernetes.io/cluster/bookstore" = "shared"
}

# NAT Gateway for private subnet outbound
enable_nat_gateway     = true
single_nat_gateway     = true   # dev: 1 NAT ($32/mo)
one_nat_gateway_per_az = false  # prod: 1 per AZ
Connects to: ALB (public), private subnets (NAT)
Local equivalent: kind NodePort mappings (extraPortMappings)
Purpose: Public subnets host the Application Load Balancer (the internet entry point) and the NAT Gateway. The Internet Gateway (IGW) provides bidirectional internet access for public subnets. The NAT Gateway allows private subnets (EKS nodes) to pull container images and access AWS APIs without being directly reachable from the internet.

4 Private Subnets (EKS Nodes)

# Inside module "vpc" { ... }

# EKS worker nodes run here — no direct internet
private_subnets = [
  "10.0.10.0/24",
  "10.0.20.0/24",
  "10.0.30.0/24"
]
private_subnet_tags = {
  # Tag for internal ALB discovery
  "kubernetes.io/role/internal-elb" = 1
  "kubernetes.io/cluster/bookstore" = "shared"
}
Connects to: EKS cluster, NAT Gateway (egress), Database subnets
Local equivalent: kind worker nodes (bookstore-worker, bookstore-worker2)
Purpose: All EKS worker nodes (both system and application node groups) run in private subnets. They have no public IP addresses and can only reach the internet outbound through the NAT Gateway. This prevents direct attack surface on compute resources. The internal-elb tag lets the ALB Controller create internal load balancers here when needed.

5 Database Subnets

# Inside module "vpc" { ... }

# Most restricted tier — no internet, SG-locked
database_subnets = [
  "10.0.100.0/24",
  "10.0.101.0/24",
  "10.0.102.0/24"
]

# Create a DB subnet group for RDS and ElastiCache
create_database_subnet_group       = true
create_database_subnet_route_table = true
database_subnet_group_name         = "bookstore-dev-db"
Connects to: Aurora PostgreSQL, ElastiCache Redis
Local equivalent: CNPG pods in K8s cluster (same network)
Purpose: Database subnets are the most isolated tier. They have no NAT Gateway route, so nothing in these subnets can reach the internet. Only traffic from the EKS node security group on port 5432 (PostgreSQL) and 6379 (Redis) is allowed. This defense-in-depth approach means even if an EKS node is compromised, database access is still limited by security groups.

6 EKS Cluster

# 04-eks.tf
module "eks" {
  source  = "terraform-aws-modules/eks/aws"
  version = "~> 20.31"

  cluster_name    = "bookstore"
  cluster_version = "1.31"

  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_subnets

  # OIDC for pod-level IAM (IRSA)
  enable_irsa = true

  cluster_addons = {
    vpc-cni            = { most_recent = true }
    coredns            = { most_recent = true }
    kube-proxy         = { most_recent = true }
    aws-ebs-csi-driver = { most_recent = true }
  }
}
Connects to: VPC private subnets, OIDC provider, node groups
Local equivalent: kind cluster control plane
Purpose: Amazon EKS manages the Kubernetes control plane (API server, etcd, scheduler, controller manager). You never patch or scale these components. The OIDC provider enables IRSA so pods can assume IAM roles without storing AWS credentials. EKS addons (VPC-CNI, CoreDNS, kube-proxy, EBS CSI) are AWS-maintained components deployed as DaemonSets and Deployments inside the cluster.

7 EKS System Node Group

# Inside module "eks" { eks_managed_node_groups ... }
system = {
  name           = "bookstore-system"
  instance_types = ["t3.medium"]  # 2 vCPU, 4 GB
  capacity_type  = "ON_DEMAND"

  # Fixed size — infra pods need predictable capacity
  min_size = 2
  max_size = 2
  desired_size = 2

  labels = {
    "nodegroup" = "system"
    "workload"  = "infrastructure"
  }

  # Only infra pods with matching toleration
  taints = [{
    key    = "workload"
    value  = "system"
    effect = "NO_SCHEDULE"
  }]
}
Connects to: EKS cluster, private subnets
Local equivalent: kind worker node 1 (bookstore-worker)
Purpose: Dedicated nodes for infrastructure workloads: Istio ztunnel, Prometheus, Grafana, Loki, Tempo, Flink, and Debezium. The NoSchedule taint prevents application pods from landing here. Fixed count (not autoscaling) ensures infrastructure has predictable capacity regardless of application traffic spikes.

8 EKS App Node Group

# Inside module "eks" { eks_managed_node_groups ... }
app = {
  name           = "bookstore-app"
  instance_types = ["t3.large"]  # 2 vCPU, 8 GB
  capacity_type  = "ON_DEMAND"

  # Autoscaling range
  min_size     = 2
  max_size     = 5
  desired_size = 2

  labels = {
    "nodegroup" = "app"
    "workload"  = "application"
  }
  # No taint — default scheduling target
}
Connects to: EKS cluster, Aurora, MSK, ElastiCache
Local equivalent: kind worker node 2 (bookstore-worker2)
Purpose: Application workloads (ecom-service, inventory-service, UI, CSRF service, Keycloak) run on these nodes. The group scales from 2 to 5 nodes automatically based on CPU/memory pressure. No taint means all pods without a specific nodeSelector land here by default. The t3.large instances provide 8 GB RAM, enough for Spring Boot's JVM heap.

9 ECR Repositories

# 05-ecr.tf
locals {
  ecr_repositories = [
    "ecom-service", "inventory-service",
    "ui-service", "csrf-service", "flink",
  ]
}

resource "aws_ecr_repository" "services" {
  for_each = toset(local.ecr_repositories)
  name     = "bookstore/${each.key}"

  image_tag_mutability = "MUTABLE"  # IMMUTABLE in prod

  image_scanning_configuration {
    scan_on_push = true
  }
}

resource "aws_ecr_lifecycle_policy" "services" {
  for_each   = aws_ecr_repository.services
  repository = each.value.name
  # Remove untagged after 1 day, keep last 10 tagged
}
Connects to: EKS nodes (image pull), CI/CD pipeline (push)
Local equivalent: kind load docker-image from local Docker
Purpose: Private container image storage with automatic vulnerability scanning on every push. Five repositories match the five buildable services. Lifecycle policies automatically delete untagged images after 1 day and keep only the last 10 tagged versions, preventing unbounded storage cost growth. In production, immutable tags prevent accidental overwrites.

10 Aurora PostgreSQL Clusters (4x)

# 06-rds.tf
resource "aws_rds_cluster" "databases" {
  for_each = local.databases  # ecom, inventory, analytics, keycloak

  cluster_identifier = "bookstore-dev-${each.key}-db"
  engine             = "aurora-postgresql"
  engine_version     = "16.4"
  database_name      = each.value.database_name
  master_username    = each.key
  master_password    = random_password.db_passwords[each.key].result

  db_subnet_group_name   = module.vpc.database_subnet_group_name
  vpc_security_group_ids = [aws_security_group.aurora.id]

  # Serverless v2 auto-scaling
  serverlessv2_scaling_configuration {
    min_capacity = 0.5
    max_capacity = 4
  }

  # CDC: wal_level=logical via parameter group
  db_cluster_parameter_group_name =
    aws_rds_cluster_parameter_group.bookstore.name
  storage_encrypted = true
}
Connects to: EKS pods (via SG), Debezium (CDC), database subnets
Local equivalent: 4 CloudNativePG clusters (2 instances each)
Purpose: Four isolated Aurora PostgreSQL clusters maintain the same per-service database isolation as the local setup. Serverless v2 scales compute (ACUs) from 0.5 to 4 based on actual query load, so dev environments cost ~$40/month per cluster while production can burst to handle traffic spikes. The rds.logical_replication=1 parameter group enables Debezium CDC to read the PostgreSQL Write-Ahead Log.

11 Amazon MSK (Kafka)

# 07-msk.tf
resource "aws_msk_cluster" "bookstore" {
  cluster_name           = "bookstore-dev-kafka"
  kafka_version          = "3.6.0"
  number_of_broker_nodes = 3

  broker_node_group_info {
    instance_type   = "kafka.t3.small"
    client_subnets  = module.vpc.private_subnets
    security_groups = [aws_security_group.msk.id]
    storage_info {
      ebs_storage_info {
        volume_size = 100
      }
    }
  }

  encryption_info {
    encryption_in_transit {
      client_broker = "TLS_PLAINTEXT"  # TLS in prod
      in_cluster    = true
    }
  }

  client_authentication {
    sasl { iam = true }
  }
}
Connects to: Debezium (CDC), ecom-service, inventory-service, Flink
Local equivalent: Kafka pod with KRaft mode in infra namespace
Purpose: Three MSK brokers across three AZs provide fault-tolerant Kafka for CDC events (ecom-connector.public.orders, inventory-connector.public.stock) and application events (order.created, inventory.updated). Replication factor 3 with minimum in-sync replicas of 2 means any single broker can fail without data loss. IAM authentication replaces the unauthenticated plaintext used locally.

12 ElastiCache Redis

# 08-elasticache.tf
resource "aws_elasticache_replication_group" "redis" {
  replication_group_id = "bookstore-dev-redis"
  description          = "CSRF, rate-limiting, sessions"

  node_type          = "cache.t3.micro"
  num_cache_clusters = 1  # 2+ for prod Multi-AZ
  engine_version     = "7.1"

  subnet_group_name  = aws_elasticache_subnet_group.redis.name
  security_group_ids = [aws_security_group.redis.id]

  transit_encryption_enabled = true
  at_rest_encryption_enabled = true
  auth_token = random_password.redis_auth.result

  automatic_failover_enabled = false # true with 2+ nodes
}
Connects to: CSRF service, ecom-service (rate limiting)
Local equivalent: Redis pod in infra namespace
Purpose: Managed Redis for three use cases: CSRF token storage (SET/GET with TTL), Bucket4j rate-limiting token buckets, and session caching. Transit encryption (TLS) means clients connect via rediss:// protocol. Auth token prevents unauthorized access. In production, a replica node enables automatic failover with zero data loss.

13 IAM Roles + IRSA

# 09-iam.tf
module "ecom_service_irsa" {
  source = "terraform-aws-modules/iam/aws//modules/iam-role-for-service-accounts-eks"

  role_name = "bookstore-ecom-service"
  role_policy_arns = {
    secrets = aws_iam_policy.ecom_secrets_access.arn
  }
  oidc_providers = {
    main = {
      provider_arn = module.eks.oidc_provider_arn
      namespace_service_accounts = ["ecom:ecom-service"]
    }
  }
}

resource "aws_iam_policy" "ecom_secrets_access" {
  policy = jsonencode({
    Statement = [{
      Effect   = "Allow"
      Action   = ["secretsmanager:GetSecretValue"]
      Resource = [aws_secretsmanager_secret.db_passwords["ecom"].arn]
    }]
  })
}
Connects to: EKS OIDC, Secrets Manager, RDS, MSK
Local equivalent: K8s ServiceAccounts + Secrets (secretKeyRef)
Purpose: IRSA eliminates static AWS credentials inside pods. Each service gets a narrowly scoped IAM role: ecom-service can only read its own DB password, inventory-service gets MSK access for Kafka, Debezium can read all three CDC database passwords. The trust relationship ensures only specific Kubernetes ServiceAccounts in specific namespaces can assume each role.

14 Route53 + ACM Certificate

# 10-route53.tf
resource "aws_acm_certificate" "main" {
  domain_name = var.domain_name
  subject_alternative_names = [
    "*.${var.domain_name}"
  ]
  validation_method = "DNS"
}

resource "aws_route53_record" "acm_validation" {
  for_each = { for dvo in
    aws_acm_certificate.main.domain_validation_options :
    dvo.domain_name => dvo
  }
  zone_id = local.zone_id
  name    = each.value.resource_record_name
  type    = each.value.resource_record_type
  ttl     = 60
  records = [each.value.resource_record_value]
}

resource "aws_route53_record" "ui" {
  zone_id = local.zone_id
  name    = var.domain_name
  type    = "A"
  alias {
    name = kubernetes_ingress_v1.main.status[0].load_balancer[0].ingress[0].hostname
    zone_id = data.aws_lb_hosted_zone_id.main.id
    evaluate_target_health = true
  }
}
Connects to: ALB (DNS alias), ACM (validation)
Local equivalent: /etc/hosts + cert-manager self-signed CA
Purpose: Route53 provides real DNS resolution instead of /etc/hosts hacks. ACM provides free, auto-renewing wildcard TLS certificates validated via DNS CNAME records. This replaces cert-manager's self-signed CA (30-day rotation, manual trust). Browsers trust ACM certificates automatically with no trust-ca.sh step needed.

15 Application Load Balancer

# 12-k8s-resources.tf — ALB Controller
resource "helm_release" "alb_controller" {
  name       = "aws-load-balancer-controller"
  repository = "https://aws.github.io/eks-charts"
  chart      = "aws-load-balancer-controller"
  version    = "1.10.0"
  namespace  = "kube-system"

  set {
    name  = "clusterName"
    value = module.eks.cluster_name
  }
  set {
    name  = "serviceAccount.annotations.eks\\.amazonaws\\.com/role-arn"
    value = module.alb_controller_irsa.iam_role_arn
  }
  set {
    name  = "vpcId"
    value = module.vpc.vpc_id
  }
}

# K8s Ingress creates ALB with host-based routing
# myecom.net         -> ui-service:80
# api.myecom.net/ecom -> ecom-service:8080
# idp.myecom.net     -> keycloak:8080
Connects to: Public subnets, ACM cert, EKS services
Local equivalent: Istio Gateway + kind NodePort 30000
Purpose: The ALB is the internet-facing HTTPS entry point. The AWS Load Balancer Controller watches for Kubernetes Ingress resources and automatically creates/updates ALBs with ACM certificates. Host-based routing sends traffic to the correct service. This replaces the Istio Kubernetes Gateway with NodePort 30000 used locally. WAF can be attached for DDoS protection and request filtering.

  chart = "ztunnel"
  # DaemonSet on every node — HBONE tunnels
}
Connects to: All application pods (mesh overlay)
Local equivalent: Same (Istio Ambient 1.29.1 on kind)

PeerAuthentication STRICT ensures no plaintext communication. This is identical to the local kind setup, meaning the same security policies, AuthorizationPolicies, and RequestAuthentication rules work on both environments.

17 Application Services (K8s Deployments)

# 12-k8s-resources.tf — App ConfigMaps
resource "kubernetes_config_map" "ecom_config" {
  metadata {
    name      = "ecom-service-config"
    namespace = "ecom"
  }
  data = {
    # Aurora (was ecom-db.ecom.svc.cluster.local)
    DATABASE_HOST = aws_rds_cluster.databases["ecom"].endpoint
    # MSK (was kafka.infra.svc.cluster.local:9092)
    KAFKA_BROKERS = aws_msk_cluster.bookstore.bootstrap_brokers_tls
    # ElastiCache (was redis.infra.svc.cluster.local)
    REDIS_HOST = aws_elasticache_replication_group.redis.primary_endpoint_address
    REDIS_TLS  = "true"
    # Keycloak (real domain)
    KEYCLOAK_ISSUER_URI = "https://idp.${var.domain_name}/realms/bookstore"
  }
}

# Same K8s manifests, different endpoints
Connects to: Aurora, MSK, ElastiCache, Keycloak, ALB
Local equivalent: Same K8s Deployments with in-cluster Service endpoints
Purpose: The application services run as the same Kubernetes Deployments as the local setup. The only difference is environment variables: instead of pointing to in-cluster services (kafka.infra.svc, ecom-db.ecom.svc), they point to AWS managed service endpoints (MSK bootstrap brokers, Aurora writer endpoints, ElastiCache primary). This 15-Factor approach means zero application code changes between local and AWS.
