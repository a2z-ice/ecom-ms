1. What Is This Service?

The CSRF service is a gateway-level security service that sits in front of all backend services in a Kubernetes cluster. Every HTTP request passing through the Istio gateway is intercepted by this service (via Envoy's ext_authz mechanism) before reaching the backend.

Browser → Istio Gateway → csrf-service (ext_authz) → Backend Service ↑ Every request passes through here. Safe methods (GET): pass immediately. Mutations (POST/PUT/DELETE): validate everything.

It performs 6 security checks on every mutating request:

  1. Origin validation — is this request from an allowed website?
  2. JWT audience check — is this token meant for our app?
  3. Token introspection — is the user still active in Keycloak?
  4. CSRF token presence — does the request include a CSRF token?
  5. CSRF token validation — does it match what we stored in Redis?
  6. Origin binding — was this token generated from the same origin?

func parseCSV(s string) []string {
    parts := strings.Split(s, ",")           // "a,b,c" → ["a", "b", "c"]
    result := make([]string, 0, len(parts)) // Pre-allocate slice
    for _, p := range parts {                 // _ ignores the index
        p = strings.TrimSpace(p)              // Remove whitespace
        if p != "" {
            result = append(result, p)        // Add to slice
        }
    }
    return result
}

Go Syntax Deep Dive: Receivers, Interfaces & Pointers

Before reading the source code, you need to understand three Go concepts that confuse every beginner: method receivers, interfaces, and pointers.

Method Receivers: What (s *RedisStore) Means

In Go, methods are defined outside the struct, with a receiver between func and the method name:

//     receiver         method name    parameters               return values
//     ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓    ↓↓↓↓↓↓↓↓      ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓     ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
func (s *RedisStore) Generate(ctx context.Context, userID, origin string) (string, error) {
    // Inside this method, "s" is the RedisStore instance (like "this" in Java or "self" in Python)
    token := uuid.New().String()
    s.client.Set(ctx, ...)        // Access the struct's fields via "s"
    return token, nil              // Return two values: the token and no error
}

Pointer receiver (*) vs Value receiver

// POINTER receiver: method receives a pointer to the struct
// Can modify the struct's fields. Used for most methods.
func (s *RedisStore) Close() error {
    return s.client.Close()   // Can access and modify s.client
}

// VALUE receiver: method receives a COPY of the struct
// Cannot modify the original. Used for read-only methods.
func (c Claims) IsExpired() bool {
    return time.Now().Unix() > c.Exp  // Just reads c.Exp, doesn't modify
}

// RULE OF THUMB: Always use *pointer receivers unless you have a reason not to.
// In this project, every method uses pointer receivers.

Interfaces: How Go Does Polymorphism

In Java/C#, you write class RedisStore implements TokenStore. Go doesn't have this. In Go, if a type has all the methods an interface requires, it automatically satisfies that interface. No declaration needed.

// Step 1: Define the interface (what methods are needed)
type TokenStore interface {
    Generate(ctx context.Context, userID, origin string) (string, error)
    Validate(ctx context.Context, userID, token, origin string) (bool, error)
    Ping(ctx context.Context) error
    Close() error
}

// Step 2: Define a struct with the SAME methods
type RedisStore struct { ... }

func (s *RedisStore) Generate(...) (string, error) { ... }  // ✓
func (s *RedisStore) Validate(...) (bool, error)   { ... }  // ✓
func (s *RedisStore) Ping(...) error              { ... }  // ✓
func (s *RedisStore) Close() error                { ... }  // ✓

// ✓ RedisStore now AUTOMATICALLY satisfies TokenStore.
// No "implements" keyword needed. Go checks at compile time.

Why this matters: Dependency Injection & Testing

// The Handler accepts the INTERFACE, not the concrete type
type Handler struct {
    Store TokenStore     // ← Interface type, NOT *RedisStore
}

// In PRODUCTION: inject a real Redis store
h := handler.New(store.NewRedisStore(...), ...)

// In TESTS: inject a fake (miniredis) store — same interface, different implementation
h := handler.New(store.NewRedisStore(miniredis.Addr(), ...), ...)

The NoopIntrospector — simplest interface implementation possible

// The interface requires one method
type Introspector interface {
    IsActive(ctx context.Context, rawToken string) (bool, error)
}

// This struct has ZERO fields
type NoopIntrospector struct{}

// One method that always returns true — satisfies the interface
func (n *NoopIntrospector) IsActive(_ context.Context, _ string) (bool, error) {
    return true, nil
}
//  _ means "I don't use this parameter" (Go requires all params to be used or underscored)
Pattern: Noop implementations. When a feature is disabled (e.g., INTROSPECT_ENABLED=false), main.go injects a NoopIntrospector instead of a real one. The handler doesn't know the difference — it just calls h.Introspector.IsActive() which always returns true.

Multiple Return Values

// Go functions can return 2+ values. The convention is: (result, error)
func (s *RedisStore) Validate(...) (bool, error) {
    return true, nil       // valid=true, error=nil (success)
    return false, nil      // valid=false, error=nil (invalid token, not an error)
    return false, err      // valid=false, error=err (Redis failure, fail-closed)
    return true, err       // valid=true, error=err (Redis failure, fail-OPEN)
}

// Calling code checks BOTH return values:
valid, err := h.Store.Validate(ctx, userID, token, origin)
if err != nil {
    // Handle error (log it, increment metric)
}
if !valid {
    // Token is invalid — return 403
}

The internal/ directory. Go has a special rule: packages under internal/ can only be imported by code in the same module. External projects cannot import github.com/bookstore/csrf-service/internal/store. This enforces encapsulation at the module level.

Network Calls Under the Hood

This service makes 4 types of network calls. Understanding how each works at the TCP/protocol level is critical for debugging production issues.

Network Call Map

csrf-service makes these network calls: 1. Redis (TCP, port 6379) — Token CRUD, rate limiting, introspection cache 2. Keycloak (HTTP POST, port 8080) — JWT introspection (RFC 7662) 3. Inbound HTTP (port 8080) — Receives ext_authz requests from Envoy 4. Prometheus scrape (port 8080) — Prometheus GETs /metrics every 15s All within the Kubernetes cluster. No external internet calls.

1. Redis Connection: How go-redis Works

// When you create a Redis client, it does NOT connect immediately
client := redis.NewClient(&redis.Options{
    Addr:         "redis.infra.svc.cluster.local:6379",  // K8s DNS resolves to pod IP
    Password:     "CHANGE_ME",
    DB:           0,
    DialTimeout:  2 * time.Second,   // Max time to establish TCP connection
    ReadTimeout:  1 * time.Second,   // Max time waiting for Redis response
    WriteTimeout: 1 * time.Second,   // Max time sending data to Redis
    PoolSize:     10,                // Keep 10 TCP connections alive
    MinIdleConns: 2,                 // Always keep 2 connections ready
})

What happens when you call s.client.Set(...):

s.client.Set(ctx, "csrf:user-123", "token-uuid", 10*time.Minute) Under the hood: 1. go-redis checks the connection pool for an idle TCP connection • Pool has idle connection? → Reuse it (no TCP handshake) • Pool empty? → Dial new TCP connection to redis:6379 • Pool full (10 connections)? → Wait until one is returned 2. Sends Redis protocol (RESP) over TCP: *4\r\n$3\r\nSET\r\n$14\r\ncsrf:user-123\r\n$36\r\n550e8400-...\r\n$2\r\nEX\r\n$3\r\n600\r\n (This is the Redis wire protocol — plain text over TCP) 3. Waits for response: +OK\r\n 4. Returns connection to pool (ready for next call) 5. If ctx deadline exceeded at any point → cancel, return error
Connection pooling is why Redis calls are fast (~0.1ms). The TCP connection (3-way handshake, TLS if configured) is established once and reused for thousands of commands. The PoolSize: 10 means up to 10 concurrent Redis commands can execute in parallel.

What happens when Redis is down:

stored, err := s.client.Get(ctx, keyPrefix+userID).Result()
// err = "dial tcp redis.infra.svc:6379: connect: connection refused"
//
// The client tried to:
//   1. Get a connection from the pool → all dead
//   2. Dial a new TCP connection → TCP SYN sent, no SYN-ACK received
//   3. Wait DialTimeout (2s) → timeout
//   4. Retry (go-redis retries 3 times by default) → all fail
//   5. Return error
//
// Then our code decides: fail-open (return true) or fail-closed (return false)
if s.failClosed {
    return false, err   // Block the request
}
return true, err      // Allow the request through

2. Keycloak Introspection: How net/http Client Works

// Create an HTTP client with a 3-second timeout
httpClient := &http.Client{
    Timeout: 3 * time.Second,  // Covers DNS + TCP + TLS + request + response
}

What happens when you call k.callKeycloak(ctx, rawToken):

k.callKeycloak(ctx, rawToken) Under the hood: 1. Build the HTTP request: POST http://keycloak.identity.svc.cluster.local:8080/realms/bookstore/protocol/openid-connect/token/introspect Content-Type: application/x-www-form-urlencoded Authorization: Basic base64(client_id:client_secret) Body: token=eyJhbGc...&token_type_hint=access_token 2. DNS resolution (inside Kubernetes): keycloak.identity.svc.cluster.local → CoreDNS query → resolves to Keycloak pod IP (e.g., 10.244.1.50) (Cached by Go's net.Resolver for ~30s) 3. TCP handshake (3-way): csrf-service → SYN → keycloak:8080 keycloak:8080 → SYN-ACK → csrf-service csrf-service → ACK → keycloak:8080 (~0.1ms within cluster) 4. HTTP request sent over TCP connection: POST /realms/bookstore/.../token/introspect HTTP/1.1 5. Keycloak processes: • Validates client credentials (Basic Auth) • Decodes the JWT • Checks user status in database • Returns: {"active": true} or {"active": false} (~5-15ms within cluster) 6. Go parses response: json.NewDecoder(resp.Body).Decode(&result) • result.Active = true or false 7. Cache in Redis: SET introspect:a1b2c3d4e5f6g7h8 "1" EX 15 (TTL = 15 seconds) 8. Next call for same token: Redis GET → cache hit → skip Keycloak entirely (~0.1ms)

The code line by line:

func (k *KeycloakIntrospector) callKeycloak(ctx context.Context, rawToken string) (bool, error) {
    // Build form data (URL-encoded body, not JSON)
    form := url.Values{
        "token":           {rawToken},
        "token_type_hint": {"access_token"},
    }

    // Create HTTP request with context (carries deadline from ctx)
    req, err := http.NewRequestWithContext(ctx, "POST", k.introspectURL,
        strings.NewReader(form.Encode()))
    // form.Encode() = "token=eyJhbGc...&token_type_hint=access_token"
    // strings.NewReader() wraps the string as an io.Reader (required by NewRequest)

    req.Header.Set("Content-Type", "application/x-www-form-urlencoded")
    req.SetBasicAuth(k.clientID, k.clientSecret)
    // SetBasicAuth adds header: Authorization: Basic base64("csrf-introspect:secret")

    // Execute the HTTP call (DNS + TCP + send + receive)
    resp, err := k.httpClient.Do(req)
    // If ctx deadline expires during the call, Do() returns immediately with err

    defer resp.Body.Close()
    // defer = "run this when the function returns" (Go's cleanup mechanism)
    // resp.Body is an io.ReadCloser — MUST be closed to release TCP connection
    // If you forget Close(), the TCP connection leaks and the pool fills up

    // Parse JSON response
    var result struct {
        Active bool `json:"active"`   // Struct tag: maps JSON "active" to Go field
    }
    json.NewDecoder(resp.Body).Decode(&result)
    // NewDecoder streams from Body (doesn't read all into memory first)
    // &result = pointer to result (Decode needs to modify it)

    return result.Active, nil
}

3. Inbound HTTP: How ext_authz Requests Arrive

How a browser request reaches csrf-service: Browser sends HTTPS request: POST https://api.service.net:30000/ecom/cart Authorization: Bearer eyJhbGc... X-CSRF-Token: 550e8400-... Kind NodePort (30000) → forwards to Istio Gateway pod Istio Gateway (Envoy): 1. TLS termination (decrypts HTTPS → plain HTTP internally) 2. JWT signature validation (RequestAuthentication CRD) 3. Sees AuthorizationPolicy CUSTOM action for csrf-ext-authz 4. Makes internal HTTP call to csrf-service: // Envoy sends this to csrf-service:8080 (cluster-internal) POST http://csrf-service.infra.svc.cluster.local:8080/ Headers forwarded: :method = POST // Original method :path = /ecom/cart // Original path authorization = Bearer eyJhbGc... // From includeRequestHeadersInCheck x-csrf-token = 550e8400-... // From includeRequestHeadersInCheck origin = https://myecom.net:30000 // From includeRequestHeadersInCheck referer = https://myecom.net:... // From includeRequestHeadersInCheck csrf-service Go HTTP server: 1. http.ServeMux matches "/" route → h.ExtAuthzCheck 2. Runs all 6 security checks 3. Returns 200 OK (allow) or 403 Forbidden (deny) Envoy reads csrf-service response: 200 → forwards original request to backend (ecom-service) 403 → returns 403 to browser (request never reaches backend)

How Go's HTTP server handles this:

// In main.go:
srv := &http.Server{
    Addr:         ":8080",         // Listen on all interfaces, port 8080
    Handler:      mux,             // Route requests to our handlers
    ReadTimeout:  5 * time.Second,  // Max time to read entire request
    WriteTimeout: 10 * time.Second, // Max time to write response
    IdleTimeout:  60 * time.Second, // Keep-alive timeout for idle connections
}

go func() {
    srv.ListenAndServe()  // Blocks forever, handling requests
}()
//
// Under the hood, ListenAndServe():
//   1. Calls net.Listen("tcp", ":8080") — binds to port, starts accepting
//   2. For each incoming TCP connection:
//      a. Spawns a NEW goroutine (lightweight thread, ~2KB stack)
//      b. Reads HTTP request from the connection
//      c. Calls mux.ServeHTTP(w, r) to route to the right handler
//      d. Handler writes response to w (http.ResponseWriter)
//      e. Connection kept alive for next request (HTTP keep-alive)
//
// KEY INSIGHT: Each request runs in its own goroutine.
// Go can handle 10,000+ concurrent requests because goroutines are cheap.
// (Java threads: ~1MB stack each. Go goroutines: ~2KB stack each.)

4. Context: How Timeouts and Cancellation Propagate

Every network call in this service accepts a context.Context as its first parameter. Context carries deadlines and cancellation signals across API boundaries.

// Create a context that expires in 2 seconds
ctx, cancel := context.WithTimeout(r.Context(), 2*time.Second)
defer cancel()  // Always call cancel to release resources

// Pass ctx to Redis call
valid, err := h.Store.Validate(ctx, userID, token, origin)
//
// If 2 seconds pass before Redis responds:
//   • ctx.Done() channel closes
//   • go-redis detects the closed context
//   • Cancels the in-flight TCP read/write
//   • Returns err = "context deadline exceeded"
//
// WHY THIS MATTERS:
// Without context, a slow Redis call could block the goroutine forever.
// With context, we guarantee every network call completes within 2s.
// This prevents a single slow dependency from crashing the entire service.

Context inheritance chain in a single request:

Envoy sends request → Go creates r.Context() (base context) └→ context.WithTimeout(r.Context(), 3s) — introspection timeout └→ passed to k.Introspector.IsActive(intrCtx, token) └→ passed to k.redis.Get(intrCtx, cacheKey) — Redis cache check └→ passed to k.httpClient.Do(req) — Keycloak HTTP call └→ context.WithTimeout(r.Context(), 2s) — CSRF validation timeout └→ passed to h.Store.Validate(ctx, ...) — Redis token check If Envoy cancels the request (e.g., client disconnected): r.Context().Done() closes → ALL child contexts cancel → ALL in-flight Redis/HTTP calls abort immediately. No wasted work.
Go's context is like a tree of deadlines. When a parent context cancels, all children cancel too. This is how Go prevents resource leaks in microservices — every goroutine and every network call has a bounded lifetime.

7. store/redis.go — Interfaces & Redis

This is the most important Go concept in the project: interfaces.

Go interfaces are implicit. A type satisfies an interface simply by implementing its methods — no implements keyword needed. If RedisStore has Generate(), Validate(), Ping(), and Close() methods, it automatically satisfies TokenStore.
// The INTERFACE — defines what operations are available
type TokenStore interface {
    Generate(ctx context.Context, userID, origin string) (string, error)
    Validate(ctx context.Context, userID, token, origin string) (bool, error)
    Ping(ctx context.Context) error
    Close() error
}

// The IMPLEMENTATION — uses Redis
type RedisStore struct {
    client     *redis.Client     // Redis connection (pointer to avoid copying)
    ttl        time.Duration     // Token expiration time
    failClosed bool              // Reject on Redis errors?
}

Line-by-line: Generate

func (s *RedisStore) Generate(ctx context.Context, userID, origin string) (string, error) {
    token := uuid.New().String()              // Generate UUID v4: "550e8400-e29b-..."

    value := token                            // Default: just the token
    if origin != "" {
        value = token + "|" + origin          // Bind to origin: "550e...|https://myecom.net:30000"
    }

    // Redis SET with TTL: key="csrf:user-123", value="uuid|origin", expires in 10min
    if err := s.client.Set(ctx, keyPrefix+userID, value, s.ttl).Err(); err != nil {
        if s.failClosed {
            return "", err                  // Fail-closed: return empty token
        }
        return token, err                     // Fail-open: return token anyway
    }
    return token, nil                         // Success: return token, no error
}

Line-by-line: Validate (single-use + timing-safe)

func (s *RedisStore) Validate(ctx context.Context, userID, token, origin string) (bool, error) {
    // Read token from Redis
    stored, err := s.client.Get(ctx, keyPrefix+userID).Result()
    if err != nil {
        if err == redis.Nil {
            return false, nil              // Key doesn't exist — invalid token
        }
        // Redis error — fail-open or fail-closed
        if s.failClosed { return false, err }
        return true, err                   // Fail-open: allow through
    }

    // Parse "token|origin" format
    storedToken := stored
    storedOrigin := ""
    if idx := strings.Index(stored, "|"); idx >= 0 {
        storedToken = stored[:idx]             // Everything before |
        storedOrigin = stored[idx+1:]          // Everything after |
    }

    // TIMING-SAFE comparison (prevents side-channel attacks)
    valid := subtle.ConstantTimeCompare([]byte(storedToken), []byte(token)) == 1

    // Also check origin binding
    if valid && storedOrigin != "" && origin != "" {
        valid = subtle.ConstantTimeCompare([]byte(storedOrigin), []byte(origin)) == 1
    }

    if valid {
        s.client.Del(ctx, keyPrefix+userID)    // SINGLE-USE: delete after first use
    }
    return valid, nil
}

8. jwt/extract.go — JWT Parsing

A JWT (JSON Web Token) has three parts separated by dots: header.payload.signature. Each part is base64url-encoded. This package decodes the payload to read claims like sub (user ID) and aud (audience).

// Custom type for handling "aud" which can be string OR array in JWTs
type Audience []string

func (a *Audience) UnmarshalJSON(data []byte) error {
    // Try parsing as a single string first: "aud": "ui-client"
    var s string
    if err := json.Unmarshal(data, &s); err == nil {
        *a = Audience{s}                       // Wrap in slice: ["ui-client"]
        return nil
    }
    // Try parsing as array: "aud": ["ui-client", "account"]
    var arr []string
    if err := json.Unmarshal(data, &arr); err != nil {
        return err
    }
    *a = Audience(arr)
    return nil
}
Go concept: custom JSON unmarshaling. By implementing the UnmarshalJSON method, you tell Go's JSON decoder how to handle a field. This is how you handle ambiguous JSON types (RFC 7519 allows aud to be either a string or array).

9. origin/validator.go — Request Validation

type Validator struct {
    allowed map[string]bool   // Hash set of allowed origins
    require bool              // Reject if Origin header missing?
}

// Result pattern: return a struct with multiple fields instead of multiple returns
type Result struct {
    Allowed bool
    Origin  string
    Missing bool
}
Go maps as sets. Go doesn't have a built-in Set type. The pattern map[string]bool is used instead — lookup is O(1) and checking membership is just m["key"] (returns false if not present).

10. ratelimit/limiter.go — Rate Limiting

This implements a sliding window rate limiter using Redis INCR + EXPIRE:

func (l *RedisLimiter) Allow(ctx context.Context, userID string) (bool, error) {
    bucket := time.Now().Truncate(time.Minute).Unix()  // Current minute as number
    key := fmt.Sprintf("ratelimit:csrf:%s:%d", userID, bucket)

    count, err := l.client.Incr(ctx, key).Result()     // Redis INCR: atomic +1
    if count == 1 {
        l.client.Expire(ctx, key, 2*time.Minute)       // Set TTL on first use
    }

    return count <= int64(l.maxPerMin), nil          // Under limit?
}

11. introspect/client.go — HTTP Client & Caching

This package makes HTTP calls to Keycloak and caches results in Redis. It demonstrates Go's net/http client, crypto/sha256 hashing, and the interface pattern.

// Cache key = SHA-256 hash of the token (first 16 hex chars)
func tokenHash(token string) string {
    h := sha256.Sum256([]byte(token))    // Hash the token
    return hex.EncodeToString(h[:8])      // Take first 8 bytes = 16 hex chars
}

13. handler/ — HTTP Handlers

The Handler Struct (Dependency Injection)

type Handler struct {
    Store            store.TokenStore          // Interface, not concrete type!
    Metrics          *middleware.Metrics
    Origin           *origin.Validator
    RateLimiter      ratelimit.Limiter         // Interface!
    Introspector     introspect.Introspector   // Interface!
    AllowedAudiences []string
    ValidateAudience bool
}
Why interfaces for dependencies? In tests, you inject miniredis (fake Redis) instead of real Redis, and NoopIntrospector instead of real Keycloak. The handler doesn't know the difference — it just calls the interface methods.

The ext_authz Decision Flow (authz.go)

ExtAuthzCheck(request): 1. Safe method (GET/HEAD)? → return 200 2. Check Origin header against allowed list • Unknown origin? → return 403 3. Extract JWT claims (base64 decode, no sig check) • No JWT? → return 200 (let backend handle 401) 4. Validate JWT audience (opt-in) • Wrong audience? → return 403 5. Introspect token with Keycloak (check cache first) • User disabled? → return 403 "Token is no longer valid" 6. Read X-Csrf-Token header • Missing? → return 403 7. Validate token in Redis (constant-time compare + origin check) • Invalid? → return 403 • Valid? → DELETE from Redis (single-use) 8. → return 200 (allow request through to backend)

14. main.go — Wiring Everything Together

func main() {
    // 1. Structured JSON logging
    slog.SetDefault(slog.New(slog.NewJSONHandler(os.Stdout, nil)))

    // 2. Load config from environment
    cfg := config.Load()

    // 3. Create all dependencies
    tokenStore := store.NewRedisStore(cfg.RedisAddr, cfg.RedisPassword, cfg.TokenTTL, cfg.FailClosed)
    metrics := middleware.NewMetrics()
    originValidator := origin.NewValidator(cfg.AllowedOrigins, cfg.RequireOrigin)
    rateLimiter := ratelimit.NewRedisLimiter(redisClient, cfg.RateLimitPerMin)

    // 4. Inject ALL dependencies into the handler
    h := handler.New(tokenStore, metrics, originValidator, rateLimiter, introspector,
        cfg.AllowedAudiences, cfg.ValidateAudience)

    // 5. Wire HTTP routes (Go 1.22+ pattern-based routing)
    mux := http.NewServeMux()
    mux.HandleFunc("GET /csrf/token", h.GenerateToken)  // Only GET
    mux.HandleFunc("GET /healthz", h.Healthz)
    mux.HandleFunc("GET /livez", h.Livez)
    mux.Handle("GET /metrics", promhttp.Handler())
    mux.HandleFunc("/", h.ExtAuthzCheck)                // Catch-all for ext_authz

    // 6. Start HTTP server in a goroutine (background)
    go func() {
        srv.ListenAndServe()   // Blocks until error or shutdown
    }()

    // 7. Wait for SIGTERM or SIGINT (Kubernetes sends SIGTERM on pod deletion)
    sigCtx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)
    <-sigCtx.Done()            // Blocks here until signal received

    // 8. Graceful shutdown: drain connections for 10 seconds
    shutdownCtx, _ := context.WithTimeout(context.Background(), 10*time.Second)
    srv.Shutdown(shutdownCtx)  // Stops accepting new requests, waits for in-flight
    tokenStore.Close()         // Close Redis connections
}
