1. NSG Architecture Overview

Network Security Groups (NSGs) are Azure's stateful Layer-4 firewalls. They filter traffic based on 5-tuple rules (source IP, source port, destination IP, destination port, protocol) and are applied to either subnets or individual NICs.

Security Architecture Diagram

Key Concepts

  • Priority-based evaluation: Rules are evaluated lowest-number-first. Priority 100 is checked before 200. First matching rule wins.
  • Stateful: If outbound traffic is allowed, the return inbound traffic is automatically permitted (no explicit return rule needed).
  • Default rules (cannot be deleted, priority 65000+):
    • AllowVNetInBound (65000) — allows all traffic within the VNet
    • AllowAzureLoadBalancerInBound (65001) — allows Azure health probes
    • DenyAllInBound (65500) — denies everything else
  • Subnet-level vs NIC-level: We apply NSGs at the subnet level for consistent enforcement. NIC-level NSGs add complexity without benefit in most AKS scenarios.
Important: The default AllowVNetInBound rule (priority 65000) allows all intra-VNet traffic. Our explicit deny rules at lower priorities override this default, enforcing micro-segmentation between subnets.

2. AKS Subnet NSG

The AKS subnet hosts all Kubernetes worker nodes. It receives traffic from the App Gateway (user requests), communicates with the database subnet (PostgreSQL), Redis subnet, and the AKS control plane (Azure-managed). All other inbound traffic is denied.

resource "azurerm_network_security_group" "aks" {
  name                = "bookstore-aks-nsg"
  location            = var.location
  resource_group_name = azurerm_resource_group.main.name
  tags                = local.common_tags

  # ── INBOUND RULES ──

  # Allow App Gateway → AKS on HTTP/HTTPS/NodePorts
  # App Gateway health probes + user traffic reach AKS pods via these ports.
  # Without this: users cannot reach the application at all.
  security_rule {
    name                       = "AllowAppGatewayInbound"
    priority                   = 100
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_ranges    = ["80", "443", "30000-32767"]
    source_address_prefix      = "10.0.30.0/24"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow AKS control plane → worker nodes (API server callbacks)
  # The AKS control plane (managed by Azure) needs to reach kubelets on
  # port 443 and 10250 for pod exec, logs, metrics, and webhook calls.
  # Without this: kubectl exec/logs fail, admission webhooks break.
  security_rule {
    name                       = "AllowAKSControlPlane"
    priority                   = 110
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_ranges    = ["443", "10250"]
    source_address_prefix      = "AzureCloud"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow intra-AKS communication (pod-to-pod, node-to-node)
  # Kubernetes pods use overlay networking within the AKS subnet.
  # Istio sidecar proxies, CoreDNS, and kube-proxy all need this.
  # Without this: all pod-to-pod and service-to-service calls fail.
  security_rule {
    name                       = "AllowAKSIntraSubnet"
    priority                   = 120
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow Azure Load Balancer health probes
  # Azure internal load balancers (used by AKS services of type
  # LoadBalancer) send probes from this well-known service tag.
  # Without this: AKS internal LB marks all backends unhealthy.
  security_rule {
    name                       = "AllowAzureLBProbes"
    priority                   = 130
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "AzureLoadBalancer"
    destination_address_prefix = "10.0.1.0/22"
  }

  # DENY all other inbound traffic
  # This overrides the default AllowVNetInBound (priority 65000),
  # blocking any traffic from subnets not explicitly permitted above
  # (e.g., Redis subnet cannot initiate connections to AKS).
  security_rule {
    name                       = "DenyAllInbound"
    priority                   = 4000
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  # ── OUTBOUND RULES ──

  # Allow AKS → Database subnet (PostgreSQL)
  # All four PostgreSQL Flexible Servers (ecom, inventory, analytics,
  # keycloak) reside in the DB subnet on port 5432.
  security_rule {
    name                       = "AllowAKSToPostgres"
    priority                   = 100
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "5432"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "10.0.8.0/24"
  }

  # Allow AKS → Redis subnet (TLS port 6380)
  # Azure Cache for Redis Premium uses port 6380 for TLS connections.
  # Used by: CSRF token store, session cache, rate limiting buckets.
  security_rule {
    name                       = "AllowAKSToRedis"
    priority                   = 110
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "6380"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "10.0.40.0/24"
  }

  # Allow AKS → Internet (container image pulls, Azure APIs, OIDC)
  # Required for: ACR image pulls, Azure AD token exchange, Keycloak
  # JWKS endpoint, OTel collector export, NTP, DNS.
  # Consider replacing with Azure Firewall for FQDN filtering.
  security_rule {
    name                       = "AllowAKSToInternet"
    priority                   = 200
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_ranges    = ["443", "80"]
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "Internet"
  }

  # Allow AKS → AKS (intra-subnet, same as inbound rule)
  security_rule {
    name                       = "AllowAKSIntraSubnetOutbound"
    priority                   = 210
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow DNS resolution (Azure DNS at 168.63.129.16)
  security_rule {
    name                       = "AllowDNS"
    priority                   = 220
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "53"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "168.63.129.16/32"
  }

  # DENY all other outbound traffic
  # Prevents AKS from reaching unexpected destinations (e.g., crypto
  # mining pools, C2 servers if a pod is compromised).
  security_rule {
    name                       = "DenyAllOutbound"
    priority                   = 4000
    direction                  = "Outbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }
}

3. Database Subnet NSG

The database subnet is the crown jewel — it contains all four PostgreSQL Flexible Server instances (ecom, inventory, analytics, keycloak). This NSG enforces the smallest possible blast radius: only the AKS subnet can reach port 5432, and nothing else.

Critical: Databases must NEVER be accessible from the internet, the App Gateway subnet, or the Redis subnet. A compromised App Gateway should not give direct database access. 
resource "azurerm_network_security_group" "db" {
  name                = "bookstore-db-nsg"
  location            = var.location
  resource_group_name = azurerm_resource_group.main.name
  tags                = local.common_tags

  # ── INBOUND RULES ──

  # Allow AKS → PostgreSQL on port 5432 ONLY
  # Application pods (ecom-service, inventory-service, keycloak,
  # analytics-flink) connect to their respective databases here.
  # This is the ONLY inbound rule — maximum isolation.
  security_rule {
    name                       = "AllowAKSToPostgres"
    priority                   = 100
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "5432"
    source_address_prefix      = "10.0.1.0/22"
    destination_address_prefix = "10.0.8.0/24"
  }

  # DENY from App Gateway subnet
  # Explicit deny at higher priority than VNet default.
  # Prevents: If App GW is compromised (e.g., WAF bypass),
  # attacker cannot pivot directly to the database.
  security_rule {
    name                       = "DenyAppGatewayToDb"
    priority                   = 200
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "10.0.30.0/24"
    destination_address_prefix = "10.0.8.0/24"
  }

  # DENY from Redis subnet
  # Redis has no business talking to PostgreSQL.
  security_rule {
    name                       = "DenyRedisToDb"
    priority                   = 210
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "10.0.40.0/24"
    destination_address_prefix = "10.0.8.0/24"
  }

  # DENY from Internet
  # Defense in depth: even if public_network_access is misconfigured,
  # the NSG blocks internet-originating traffic at Layer 4.
  security_rule {
    name                       = "DenyInternetToDb"
    priority                   = 300
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "Internet"
    destination_address_prefix = "10.0.8.0/24"
  }

  # DENY all other inbound
  security_rule {
    name                       = "DenyAllInbound"
    priority                   = 4000
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  # ── OUTBOUND RULES ──

  # Allow DB → AKS (response traffic, replication)
  # PostgreSQL needs to respond to AKS queries.
  # NSGs are stateful so return traffic is auto-allowed,
  # but this explicit rule covers any DB-initiated connections
  # (e.g., pg_notify, logical replication to Debezium).
  security_rule {
    name                       = "AllowDbToAKS"
    priority                   = 100
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "10.0.8.0/24"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow DB → Azure services (backups, metrics, HA replication)
  # PostgreSQL Flexible Server needs to reach Azure Storage for
  # automated backups and Azure Monitor for diagnostics.
  security_rule {
    name                       = "AllowDbToAzureServices"
    priority                   = 110
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "443"
    source_address_prefix      = "10.0.8.0/24"
    destination_address_prefix = "AzureCloud"
  }

  # DENY all other outbound
  security_rule {
    name                       = "DenyAllOutbound"
    priority                   = 4000
    direction                  = "Outbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }
}

4. App Gateway Subnet NSG

The App Gateway subnet hosts the Azure Application Gateway v2. It is the only component directly exposed to the internet. This NSG allows user traffic on 443/80 and the mandatory GatewayManager health probe range.

Mandatory: Ports 65200-65535 from the GatewayManager service tag MUST be allowed. Without this rule, Azure cannot provision or update the App Gateway, and it enters a "Failed" state.

  # ── INBOUND RULES ──

  # Allow HTTPS from internet (user traffic)
  # This is the front door. TLS terminates at App Gateway,
  # which performs WAF inspection before forwarding to AKS.
  security_rule {
    name                       = "AllowHTTPSFromInternet"
    priority                   = 100
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "443"
    source_address_prefix      = "Internet"
    destination_address_prefix = "10.0.30.0/24"
  }

  # Allow HTTP from internet (redirect to HTTPS)
  # App Gateway listens on 80 solely to issue a 301 redirect
  # to the HTTPS URL. No application traffic flows over HTTP.
  security_rule {
    name                       = "AllowHTTPFromInternet"
    priority                   = 110
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "80"
    source_address_prefix      = "Internet"
    destination_address_prefix = "10.0.30.0/24"

  # communicate with the App Gateway control plane. If blocked,
  # the App Gateway enters a "Failed" provisioning state and
  # cannot be created, updated, or scaled.

    name                       = "AllowGatewayManager"

    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "65200-65535"
    source_address_prefix      = "GatewayManager"
    destination_address_prefix = "10.0.30.0/24"
  }

  # Allow Azure Load Balancer health probes
  security_rule {
    name                       = "AllowAzureLBProbes"
    priority                   = 130
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "AzureLoadBalancer"
    destination_address_prefix = "10.0.30.0/24"
  }

  # DENY all other inbound
  security_rule {
    name                       = "DenyAllInbound"
    priority                   = 4000
    direction                  = "Inbound"
    access                     = "Deny"
    protocol                   = "*"
    source_port_range          = "*"
    destination_port_range     = "*"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }

  # ── OUTBOUND RULES ──

  # Allow App GW → AKS (forward user traffic to backend pods)
  # App Gateway routes requests to AKS NodePorts (30000-32767)
  # and standard HTTP/HTTPS ports for health probes.
  security_rule {
    name                       = "AllowAppGwToAKS"
    priority                   = 100
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_ranges    = ["80", "443", "30000-32767"]
    source_address_prefix      = "10.0.30.0/24"
    destination_address_prefix = "10.0.1.0/22"
  }

  # Allow App GW → Azure (OCSP, CRL checks, Azure Monitor)
  security_rule {
    name                       = "AllowAppGwToAzure"
    priority                   = 110
    direction                  = "Outbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "443"
    source_address_prefix      = "10.0.30.0/24"

  # DENY App GW → Database subnet (no direct DB access)

6. Private Endpoints for All PaaS Services

Private Endpoints project an Azure PaaS service into your VNet via a private IP address. This means traffic between AKS and the PaaS service stays entirely within the Azure backbone — it never traverses the public internet. Combined with disabling public access, this creates a fully private data plane.

How Private Endpoints work: Azure creates a NIC in your VNet with a private IP. A Private DNS Zone (e.g., privatelink.postgres.database.azure.com) maps the service's FQDN to this private IP, so applications resolve the same hostname but get a private IP instead of a public one.
