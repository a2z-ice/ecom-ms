Enhancement 1: Single-Use CSRF Tokens

1 Token consumed after successful mutation

After a mutating request (POST/PUT/DELETE/PATCH) successfully validates the CSRF token, the token is deleted from Redis instead of having its TTL refreshed. The next mutation triggers the UI's auto-retry: 403 → fetch new token → retry → success.

// BEFORE: Refresh TTL (multi-use)
s.client.Expire(ctx, key, s.ttl)

// AFTER: Delete key (single-use)
s.client.Del(ctx, key)
Critical: Single-use alone is not enough. If a token is stolen but the user never performs a mutation, the stolen token sits valid until TTL expires (10min). The TTL + origin binding are essential safety nets.

Enhancement 3: Origin/Referer Validation

3 Stolen JWT blocked from unknown origins

Validates Origin header (fallback to Referer) against allowed list. Mismatch → 403.

Env: CSRF_ALLOWED_ORIGINS=https://myecom.net:30000,https://localhost:30000

Env: CSRF_REQUIRE_ORIGIN=true to reject when both headers are missing.

Enhancement 4: Token Binding to Origin

4 Token tied to the origin where it was generated

Stored as {token}|{origin} in Redis. Validation checks both. Exfiltrated token only works from the same origin.

Backward compatible: pre-upgrade tokens (no |) work from any origin.

Enhancement 5: Per-User Rate Limiting

5 Prevent token generation spam (targeted DoS)

Without rate limiting, an attacker with a stolen JWT can overwrite the victim's token 1000x/second — victim can never complete a mutation.

Sliding window: INCR + EXPIRE on Redis key ratelimit:csrf:{userId}:{minute}

Default: 10 per minute. Response: 429 Too Many Requests with Retry-After: 60

Env: CSRF_RATE_LIMIT=10

Enhancement 6: JWT Audience Validation

6 Prevent token confusion attacks

Opt-in validation of JWT aud claim. Handles both string and array per RFC 7519.

Env: CSRF_VALIDATE_AUDIENCE=true, CSRF_ALLOWED_AUDIENCES=ui-client

Enhancement 7: Open Redirect Fix + Security Headers

7 Prevent auth token theft via redirect

CallbackPage.tsx now validates returnUrl against a whitelist. Unknown origins redirect to /.

New nginx headers: Strict-Transport-Security, X-Permitted-Cross-Domain-Policies, CSP base-uri + form-action.

Defense Layering

Request → TLS termination → JWT validation (Istio)
    → Origin validation (csrf-service)
    → Audience validation (csrf-service, opt-in)
    → CSRF token validation (csrf-service)
        → Token consumed on success (single-use)
        → Token bound to origin
        → Token expires after TTL (10min max, sliding)
    → Rate limiting on token generation
    → Forward to backend
Each layer catches a different attack vector. No single mechanism is sufficient alone. TTL is the safety net for stolen-but-unconsumed tokens.
