JWT Introspection on Mutations

When an admin blocks a user, the JWT remains valid until expiration. This enhancement adds real-time token validation at the gateway for state-changing operations.

68 Unit Tests RFC 7662 Compliant Redis Cached (15s TTL) 5min Token Lifespan Mutations Only

The Problem

BEFORE

Admin blocks user at 10:00 AM

User's JWT was issued at 9:45 AM with 30-minute TTL.

User continues to place orders, modify cart, and access admin panel until 10:15 AM (token expiry).

30-minute window of unauthorized access.

AFTER

Admin blocks user at 10:00 AM

Gateway introspects the JWT on the next mutation.

Keycloak returns active: false. User gets 403 within 15 seconds (cache TTL).

15-second window maximum.

What Introspection Catches

ScenarioIntrospection ResultWithout Introspection

Role change mid-sessionNOT caught (claims frozen in JWT)Same — needs token refresh

Where Introspection Fits in the Request Flow

Request → TLS termination → JWT signature (Istio RequestAuthentication) → ext_authz → csrf-service: 1 Safe method (GET)? → 200 skip all 2 Origin validation 3 Audience validation (opt-in) 4 ★ INTROSPECTION ★ → Redis cache check Cache hit? → use cached result (~0.1ms) Cache miss? → Keycloak POST /token/introspect (~5-15ms) → Cache result in Redis (15s TTL) active: false? → 403 "Token is no longer valid" 5 CSRF token validation (single-use, origin-bound) 6 → 200 (forward to backend)
Key design decision: Introspection runs BEFORE CSRF token validation. If the user is disabled, we reject immediately without consuming the CSRF token.

Code: Before vs After

New Package: introspect/client.go

type Introspector interface {
    IsActive(ctx context.Context, rawToken string) (bool, error)
}

type KeycloakIntrospector struct {
    httpClient    *http.Client
    introspectURL string
    clientID      string
    clientSecret  string
    redis         *redis.Client
    cacheTTL      time.Duration
    failOpen      bool
}

func (k *KeycloakIntrospector) IsActive(ctx context.Context, rawToken string) (bool, error) {
    cacheKey := cachePrefix + tokenHash(rawToken)

    // 1. Check Redis cache first
    cached, err := k.redis.Get(ctx, cacheKey).Result()
    if err == nil {
        return cached == "1", nil  // Cache hit: ~0.1ms
    }

    // 2. Cache miss — call Keycloak introspection
    active, err := k.callKeycloak(ctx, rawToken)
    if err != nil {
        if k.failOpen { return true, err }  // Don't block on Keycloak failure
        return false, err
    }

    // 3. Cache the result
    k.redis.Set(ctx, cacheKey, boolToString(active), k.cacheTTL)
    return active, nil
}

Keycloak Call: RFC 7662

// POST to Keycloak introspection endpoint
POST /realms/bookstore/protocol/openid-connect/token/introspect
Content-Type: application/x-www-form-urlencoded
Authorization: Basic {base64(client_id:client_secret)}

token=<jwt>&token_type_hint=access_token

// Response:
{"active": false}  ← user disabled/session terminated/token revoked
{"active": true}   ← user is valid, token is active

Redis Cache Strategy

// Cache key: SHA-256 hash of token (first 16 hex chars)
Key:   introspect:a1b2c3d4e5f6g7h8
Value: "1" (active) or "0" (inactive)
TTL:   15 seconds (configurable via INTROSPECT_CACHE_TTL_SECONDS)

// Performance: ~95% cache hit rate under normal load
// Active user: <0.5ms average (Redis GET)
// First call: 5-15ms (Keycloak round-trip within cluster)

Manual Testing Guide

1 Enable Introspection

# Set env vars on csrf-service deployment
kubectl set env deploy/csrf-service -n infra \
  INTROSPECT_ENABLED=true \
  INTROSPECT_URL=http://keycloak.identity.svc.cluster.local:8080/realms/bookstore/protocol/openid-connect/token/introspect \
  INTROSPECT_CLIENT_ID=ecom-service \
  INTROSPECT_CLIENT_SECRET=<secret>

2 Get JWT and CSRF Token

JWT=$(curl -sk -X POST "https://idp.keycloak.net:30000/realms/bookstore/protocol/openid-connect/token" \
  -d "grant_type=password&client_id=ui-client&username=user1&password=CHANGE_ME" \
  -H "Content-Type: application/x-www-form-urlencoded" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

TOKEN=$(curl -sk https://api.service.net:30000/csrf/token \
  -H "Authorization: Bearer $JWT" | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")

3 Disable User in Keycloak

ADMIN_TOKEN=$(curl -sk -X POST "https://idp.keycloak.net:30000/realms/master/protocol/openid-connect/token" \
  -d "grant_type=password&client_id=admin-cli&username=admin&password=CHANGE_ME" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

# Get user ID
USER_ID=$(curl -sk "https://idp.keycloak.net:30000/admin/realms/bookstore/users?username=user1" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | python3 -c "import sys,json; print(json.load(sys.stdin)[0]['id'])")

# Disable user
curl -sk -X PUT "https://idp.keycloak.net:30000/admin/realms/bookstore/users/$USER_ID" \
  -H "Authorization: Bearer $ADMIN_TOKEN" -H "Content-Type: application/json" \
  -d '{"enabled": false}'

4 Wait for Cache Expiry and Test

# Wait 20 seconds (cache TTL is 15s)
sleep 20

# Attempt mutation — should get 403
curl -sk -w "%{http_code}\n" -X POST https://api.service.net:30000/ecom/cart \
  -H "Authorization: Bearer $JWT" \
  -H "X-CSRF-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"bookId":"00000000-0000-0000-0000-000000000001","quantity":1}'
# Expected: 403 "Token is no longer valid"

5 Re-enable User

curl -sk -X PUT "https://idp.keycloak.net:30000/admin/realms/bookstore/users/$USER_ID" \
  -H "Authorization: Bearer $ADMIN_TOKEN" -H "Content-Type: application/json" \
  -d '{"enabled": true}'
