2. Enhancement 1: Single-Use CSRF Tokens

Problem: Stolen tokens could be replayed unlimited times. Each use extended the TTL, making the window longer.

BEFORE

store/redis.go — Validate()

// Refresh TTL on successful validation
if valid {
    s.client.Expire(ctx,
        keyPrefix+userID, s.ttl)
}
return valid, nil

Token stays in Redis. Can be used again and again. TTL resets to 10min on every safe-method request (sliding TTL).

AFTER

store/redis.go — Validate()

// Single-use: consume token
if valid {
    s.client.Del(ctx,
        keyPrefix+userID)
}
return valid, nil

Token deleted from Redis after first successful use. Next mutation gets 403 → UI auto-retries with fresh token.

How the UI Handles Single-Use Transparently

// ui/src/api/client.ts:55-57
// Auto-retry on 403 for mutating requests — CSRF token may have been consumed
if (resp.status === 403 && isMutating && !_csrfRetried) {
    await fetchCsrfToken()                    // GET /csrf/token → new token
    return request<T>(url, options, true)  // Retry original request once
}

4. Enhancement 3: Origin/Referer Validation

Problem: Stolen JWT used from any origin — curl, Postman, attacker's server. Zero defense.

BEFORE

authz.go

// No origin check at all
userID := jwt.ExtractSub(
    r.Header.Get("Authorization"))
// Jump straight to CSRF token check

Istio ext_authz config

includeRequestHeadersInCheck:
  - authorization
  - x-csrf-token
AFTER

authz.go

// Origin validation BEFORE token check
result := h.Origin.Validate(r)
if !result.Allowed {
    writeForbidden(w,
        "Request origin not allowed")
    return
}

Istio ext_authz config

includeRequestHeadersInCheck:
  - authorization
  - x-csrf-token
  - origin      # NEW
  - referer     # NEW

New Package: origin/validator.go

func (v *Validator) Validate(r *http.Request) Result {
    origin := r.Header.Get("Origin")
    // Browsers send "null" for cross-origin OIDC form submissions
    if origin == "" || origin == "null" {
        origin = ""
        // Fallback: extract scheme://host from Referer
        ref := r.Header.Get("Referer")
        if ref != "" {
            if u, err := url.Parse(ref); err == nil && u.Host != "" {
                origin = u.Scheme + "://" + u.Host
            }
        }
    }
    if origin == "" {
        return Result{Allowed: !v.require, Missing: true}  // Configurable
    }
    return Result{Allowed: v.allowed[origin], Origin: origin}
}

5. Enhancement 4: Token Binding to Origin

Problem: Exfiltrated CSRF token worked from any origin.

BEFORE

Redis storage

KEY:   csrf:<sub>
VALUE: 550e8400-e29b-41d4-...

Token works from any origin

AFTER

Redis storage

KEY:   csrf:<sub>
VALUE: 550e8400...|https://myecom.net:30000

Token bound to generation origin. Validated with ConstantTimeCompare.

// store/redis.go — Generate with origin binding
value := token
if origin != "" {
    value = token + "|" + origin   // Pipe-delimited: "uuid|https://myecom.net:30000"
}
s.client.Set(ctx, keyPrefix+userID, value, s.ttl)

// store/redis.go — Validate with origin check
storedToken, storedOrigin := parseStoredValue(stored)  // Split by "|"
valid := subtle.ConstantTimeCompare([]byte(storedToken), []byte(token)) == 1
if valid && storedOrigin != "" && origin != "" {
    valid = subtle.ConstantTimeCompare([]byte(storedOrigin), []byte(origin)) == 1
}
Backward compatible: Tokens stored without | (pre-upgrade) are treated as origin-agnostic.

6. Enhancement 5: Per-User Rate Limiting

Problem: Attacker with stolen JWT spams GET /csrf/token 1000x/sec → overwrites victim's token → victim can never complete a mutation.

BEFORE 
// No rate limiting at all
token, err := h.Store.Generate(
    ctx, userID)
// Always succeeds
AFTER 
// Rate limit check BEFORE generation
allowed, err := h.RateLimiter.Allow(
    ctx, claims.Sub)
if !allowed {
    w.WriteHeader(429)  // Too Many Requests
    return
}
token, err := h.Store.Generate(
    ctx, claims.Sub, reqOrigin)

New Package: ratelimit/limiter.go

// Redis sliding window: INCR + EXPIRE per minute bucket
func (l *RedisLimiter) Allow(ctx context.Context, userID string) (bool, error) {
    bucket := time.Now().Truncate(time.Minute).Unix()
    key := fmt.Sprintf("ratelimit:csrf:%s:%d", userID, bucket)
    count, err := l.client.Incr(ctx, key).Result()
    if err != nil {
        return true, err  // Fail-open: don't block on Redis errors
    }
    if count == 1 {
        l.client.Expire(ctx, key, 2*time.Minute)
    }
    return count <= int64(l.maxPerMin), nil
}

7. Enhancement 6: JWT Audience Validation

Problem: JWT from wrong Keycloak client (wrong aud claim) accepted — token confusion attacks possible.

BEFORE

jwt/extract.go

var claims struct {
    Sub string `json:"sub"`
}
// Only extracted "sub"
// No audience, no issuer
AFTER

jwt/extract.go

type Claims struct {
    Sub string   `json:"sub"`
    Aud Audience `json:"aud"`
    Iss string   `json:"iss"`
}
// Audience handles string or []string
// per RFC 7519
// jwt/extract.go — ValidateAudience
func ValidateAudience(claims Claims, allowed []string) bool {
    for _, claimAud := range claims.Aud {
        for _, a := range allowed {
            if claimAud == a { return true }
        }
    }
    return false
}

10. Architecture: Enhanced Request Flow

Request → TLS termination (Envoy) → JWT signature validation (Istio RequestAuthentication) → ext_authz → csrf-service: 1. Safe method? → 200 (pass) 2. Origin validation — check Origin/Referer against allowed list Unknown origin? → 403 3. JWT audience check (opt-in) — validate aud claim Wrong audience? → 403 4. Missing CSRF header? → 403 5. Token validation — Redis GET, ConstantTimeCompare + origin binding — compare stored origin Invalid? → 403 Redis error? → 503 (fail-closed) or 200 (fail-open) 6. Token consumed (DEL from Redis) — single-use 7. → 200 (allow) — forward to backend → Backend service processes request

11. Defense Layering Diagram

If token is stolen AND... Defense that fires ────────────────────────────────────────────────────────── Attacker uses from wrong origin → Origin binding blocks (403) User already used token → Single-use blocks (consumed) User hasn't used it yet → TTL expires (10min max) Attacker spams token generation → Rate limiting (60/min) JWT from wrong Keycloak client → Audience validation (403) Redirect to attacker's site → Origin whitelist blocks No single mechanism is sufficient alone. TTL is the safety net for stolen-but-unconsumed tokens.
