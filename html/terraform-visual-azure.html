1 Azure Blob Storage Backend

00-backend.tf

terraform {
  backend "azurerm" {
    # Storage account for remote state
    resource_group_name  = "bookstore-tfstate-rg"
    storage_account_name = "bookstoretfstate"
    container_name       = "tfstate"

    # State file name per environment
    key = "bookstore.terraform.tfstate"

    # Azure CLI auth (inherits az login)
    use_azuread_auth = true
  }
}
Purpose: Remote state with lease-based locking prevents concurrent applies from corrupting state. Shared across all team members and CI/CD pipelines.
Connects to: Every terraform plan/apply reads and writes this state file.
Maps to local: terraform.tfstate file on disk (no locking, single developer).

2 Resource Group

03-resource-group.tf

locals {
  common_tags = merge({
    project     = var.project
    environment = var.environment
    managed_by  = "terraform"
    created_at  = timestamp()
  }, var.tags)

  name_prefix = "${var.project}-${var.environment}"
}

resource "azurerm_resource_group" "main" {
  name     = var.resource_group_name
  location = var.location
  tags     = local.common_tags
}
Purpose: Logical container for all Azure resources. Simplifies cost tracking and teardown: deleting the group removes everything inside it.
Connects to: Every other resource references azurerm_resource_group.main.
Maps to local: No direct equivalent. The kind cluster manages all resources implicitly.

3 Virtual Network (VNet)

04-vnet.tf

resource "azurerm_virtual_network" "main" {
  name                = "${local.name_prefix}-vnet"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  address_space       = var.vnet_address_space
  tags                = local.common_tags
}

# 4 subnets defined separately:
# - AKS:   10.0.0.0/22  (Azure CNI pod IPs)
# - DB:    10.0.10.0/24 (PostgreSQL delegation)
# - Redis: 10.0.20.0/24 (private endpoint)
# - AppGW: 10.0.30.0/24 (dedicated for App GW)
Purpose: Isolated network boundary for all resources. Azure CNI integration gives every pod a VNet-routable IP, enabling direct connectivity to managed services without NAT.
Connects to: Contains all 4 subnets. NSGs attach to each subnet for L4 filtering.
Maps to local: Docker bridge network created by kind. All containers share the same flat network.

4 AKS Subnet (/22)

04-vnet.tf (AKS subnet)

resource "azurerm_subnet" "aks" {
  name                 = "${local.name_prefix}-aks-subnet"
  resource_group_name  = azurerm_resource_group.main.name
  virtual_network_name = azurerm_virtual_network.main.name
  address_prefixes     = [var.aks_subnet_cidr]

  # Service endpoints for VNet-direct access
  service_endpoints = [
    "Microsoft.Sql",
    "Microsoft.Storage",
    "Microsoft.EventHub",
    "Microsoft.KeyVault",
  ]
}
Purpose: Azure CNI assigns a VNet IP to every pod, so a /24 (254 IPs) would be exhausted quickly. The /22 provides 1022 usable IPs for nodes and pods. Service endpoints route traffic to managed services over the Azure backbone.
Connects to: AKS nodes, AKS pods, Event Hubs, Key Vault, Storage.
Maps to local: Kind node network. All kind containers share a Docker bridge with auto-assigned IPs.

5 Database Subnet (Delegated)

04-vnet.tf (Database subnet)

resource "azurerm_subnet" "database" {
  name                 = "${local.name_prefix}-db-subnet"
  resource_group_name  = azurerm_resource_group.main.name
  virtual_network_name = azurerm_virtual_network.main.name
  address_prefixes     = [var.db_subnet_cidr]

  # PostgreSQL requires service delegation
  delegation {
    name = "postgresql-delegation"
    service_delegation {
      name = "Microsoft.DBforPostgreSQL/flexibleServers"
      actions = [
        "Microsoft.Network/virtualNetworks/subnets/join/action",
      ]
    }
  }
}
Purpose: PostgreSQL Flexible Server requires a dedicated subnet with service delegation. No other resources can share this subnet. The private DNS zone resolves database FQDNs to private IPs within the VNet.
Connects to: 4 PostgreSQL Flexible Servers, Private DNS Zone linked to VNet.
Maps to local: CloudNativePG pods running in the kind cluster (ecom-db, inventory-db, analytics-db, keycloak-db).

6 Application Gateway Subnet

04-vnet.tf (App Gateway subnet)

resource "azurerm_subnet" "appgw" {
  name                 = "${local.name_prefix}-appgw-subnet"
  resource_group_name  = azurerm_resource_group.main.name
  virtual_network_name = azurerm_virtual_network.main.name
  address_prefixes     = [var.appgw_subnet_cidr]
}

# App Gateway v2 requires its own dedicated
# subnet. Must not contain any other resources.
Purpose: Azure Application Gateway v2 mandates a dedicated subnet with no other resources. This subnet hosts the gateway instances that handle HTTPS termination and routing.
Connects to: Application Gateway, Public IP, routes traffic to AKS subnet.
Maps to local: Kind NodePort on port 30000 with Istio Gateway TLS termination.

7 Network Security Groups (NSGs)

04-vnet.tf (NSG rules)

resource "azurerm_network_security_group" "aks" {
  name = "${local.name_prefix}-aks-nsg"
  ...

  security_rule {
    name                   = "AllowAppGateway"
    priority               = 100
    direction              = "Inbound"
    access                 = "Allow"
    protocol               = "Tcp"
    destination_port_range = "80-443"
    source_address_prefix  = var.appgw_subnet_cidr
  }

  security_rule {
    name                   = "AllowOutboundPostgreSQL"
    priority               = 100
    direction              = "Outbound"
    destination_port_range = "5432"
    destination_address_prefix = var.db_subnet_cidr
  }
}

resource "azurerm_subnet_network_security_group_association" "aks" {
  subnet_id                 = azurerm_subnet.aks.id
  network_security_group_id = azurerm_network_security_group.aks.id
}
Purpose: L4 firewall rules per subnet. AKS NSG allows inbound from App Gateway and inter-node traffic. DB NSG allows inbound only from AKS on port 5432 with deny-all fallback. AppGW NSG allows internet HTTPS and Azure health probes.
Connects to: Each NSG is associated with its respective subnet.
Maps to local: Kubernetes NetworkPolicies per namespace (e.g., ecom-netpol, observability-netpol).

8 AKS Cluster

05-aks.tf

resource "azurerm_kubernetes_cluster" "main" {
  name                = "${local.name_prefix}-aks"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  dns_prefix          = "${local.name_prefix}-aks"
  kubernetes_version  = var.kubernetes_version

  identity { type = "SystemAssigned" }

  network_profile {
    network_plugin = "azure"     # Azure CNI
    network_policy = "calico"
    service_cidr   = "10.1.0.0/16"
    dns_service_ip = "10.1.0.10"
  }

  # Workload Identity for passwordless auth
  oidc_issuer_enabled       = true
  workload_identity_enabled = true
  azure_policy_enabled      = true

  oms_agent {
    log_analytics_workspace_id =
      azurerm_log_analytics_workspace.aks.id
  }
}
Purpose: Managed Kubernetes with Azure CNI, workload identity, and OIDC issuer. Azure manages the control plane (API server, etcd, scheduler). Calico enforces NetworkPolicy resources.
Connects to: VNet (via Azure CNI), ACR (AcrPull role), Log Analytics, node pools.
Maps to local: kind create cluster --name bookstore with control-plane + worker nodes.

9 AKS System Node Pool

05-aks.tf (default_node_pool)

default_node_pool {
  name            = "system"
  vm_size         = var.system_node_vm_size
  node_count      = var.system_node_count  # 2
  vnet_subnet_id  = azurerm_subnet.aks.id
  os_disk_size_gb = 128
  os_disk_type    = "Managed"
  type            = "VirtualMachineScaleSets"
  max_pods        = 50

  # Label so app workloads avoid this pool
  node_labels = {
    "nodepool-type" = "system"
  }

  upgrade_settings {
    max_surge = "10%"
  }
}
Purpose: Runs cluster-critical infrastructure workloads: kube-system, Istio, cert-manager, External Secrets Operator. Labeled as "system" to keep application pods on the app pool.
Connects to: Part of the AKS cluster, uses the AKS subnet for networking.
Maps to local: Kind worker node 1 (bookstore-worker).

10 AKS App Node Pool

05-aks.tf (app node pool)

resource "azurerm_kubernetes_cluster_node_pool" "app" {
  name                  = "app"
  kubernetes_cluster_id = azurerm_kubernetes_cluster.main.id
  vm_size               = var.app_node_vm_size
  min_count             = var.app_node_min_count  # 2
  max_count             = var.app_node_max_count  # 5
  auto_scaling_enabled  = true
  os_disk_size_gb       = 128
  vnet_subnet_id        = azurerm_subnet.aks.id
  max_pods              = 50

  node_labels = {
    "nodepool-type" = "app"
  }

  node_taints = []  # Accept app workloads
}
Purpose: Application workloads with autoscaling from 2 to 5 nodes. D4s_v3 (4 vCPU, 16 GB) provides headroom for rolling updates and traffic spikes.
Connects to: AKS cluster, AKS subnet. Autoscaler profile configured on the cluster resource.
Maps to local: Kind worker node 2 (bookstore-worker2). No autoscaling locally.

12 PostgreSQL Flexible Server (4x)

07-postgresql.tf

locals {
  databases = {
    ecom      = { name = "...-ecom-db",      database = "ecom" }
    inventory = { name = "...-inventory-db", database = "inventory" }
    analytics = { name = "...-analytics-db", database = "analytics" }
    keycloak  = { name = "...-keycloak-db",  database = "keycloak" }
  }
}

resource "azurerm_postgresql_flexible_server" "databases" {
  for_each = local.databases

  name                   = each.value.name
  version                = "16"
  sku_name               = var.db_sku_name
  storage_mb             = var.db_storage_mb
  administrator_login    = var.db_admin_username
  administrator_password = random_password.db_passwords[each.key].result

  # VNet integration (no public endpoint)
  delegated_subnet_id = azurerm_subnet.database.id
  private_dns_zone_id = azurerm_private_dns_zone.postgresql.id
}

# wal_level=logical for Debezium CDC
resource "azurerm_postgresql_flexible_server_configuration" "wal_level" {
  for_each = local.databases
  name     = "wal_level"
  value    = "logical"
  server_id = azurerm_postgresql_flexible_server.databases[each.key].id
}
Purpose: Four isolated PostgreSQL instances with VNet integration. Each service has its own server and credentials. wal_level=logical enables Debezium CDC on ecom and inventory databases.
Connects to: DB subnet (delegation), Private DNS Zone, Key Vault (passwords stored as secrets).
Maps to local: CloudNativePG clusters (ecom-db, inventory-db, analytics-db, keycloak-db) with 2 instances each.

16 Application Gateway

11-dns.tf (Application Gateway)

resource "azurerm_application_gateway" "main" {
  name = local.appgw_name

  sku {
    name = var.enable_waf ? "WAF_v2" : "Standard_v2"
    tier = var.enable_waf ? "WAF_v2" : "Standard_v2"
  }

  autoscale_configuration {
    min_capacity = 1
    max_capacity = 3
  }

  # HTTPS Listener
  http_listener {
    name                = "https-listener"
    frontend_port_name  = "https-port"  # 443
    protocol            = "Https"
    ssl_certificate_name = "appgw-ssl-cert"
  }

  # HTTP -> HTTPS redirect (301)
  redirect_configuration {
    redirect_type        = "Permanent"
    target_listener_name = "https-listener"
    include_path         = true
    include_query_string = true
  }

  # Route HTTPS to AKS backend
  request_routing_rule {
    rule_type                  = "Basic"
    http_listener_name         = "https-listener"
    backend_address_pool_name  = "aks-backend-pool"
  }

  ssl_policy {
    policy_name = "AppGwSslPolicy20220101S"
  }
}
Purpose: Internet-facing HTTPS entry point with TLS termination, HTTP-to-HTTPS redirect, health probes, and optional WAF v2 protection. Routes all traffic to the Istio ingress gateway in AKS.
Connects to: Public IP, App GW subnet, AKS backend pool, Key Vault (TLS cert), DNS Zone.
Maps to local: Istio Gateway + kind NodePort 30000 (HTTPS) + NodePort 30080 (HTTP redirect).

18 Istio Ambient Mesh

13-k8s-resources.tf (Istio)

resource "helm_release" "istio_base" {
  name       = "istio-base"
  repository = "https://istio-release.storage.googleapis.com/charts"
  chart      = "base"
  version    = "1.29.1"
  namespace  = "istio-system"
  create_namespace = true
}

resource "helm_release" "istio_cni" {
  chart   = "cni"
  version = "1.29.1"
  set { name = "profile"; value = "ambient" }
  depends_on = [helm_release.istio_base]
}

resource "helm_release" "istiod" {
  chart   = "istiod"
  version = "1.29.1"
  set { name = "profile"; value = "ambient" }
}

resource "helm_release" "ztunnel" {
  chart   = "ztunnel"
  version = "1.29.1"
  depends_on = [helm_release.istiod]
}
Purpose: Zero-trust mTLS between all pods without sidecar injection. ztunnel runs as a DaemonSet on each node, intercepting L4 traffic and encrypting it with mutual TLS. No application code changes required.
Connects to: All application namespaces (via ambient mesh label), istiod for control plane, istio-cni for traffic interception.
Maps to local: Same Istio Ambient Mesh 1.29.1 installed via Helm in the kind cluster.

Summary: Local to Azure Mapping

Key takeaway: The application code and Kubernetes manifests are nearly identical between local and Azure. Only the backing service endpoints change -- from cluster-internal addresses to Azure managed service FQDNs. Terraform provisions and wires all 20 components together, producing a production-ready AKS deployment with managed databases, Kafka-compatible messaging, HSM-backed secrets, and WAF-protected ingress.
